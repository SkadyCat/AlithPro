# 交付说明：修复思维导图画布高度为 0

## 源任务
`msg-2026-04-08T18-02-02-867Z.md` — 用户反馈点击思维导图卡片后只看到标题栏，画布区域为空

## 任务目标
修复思维导图编辑器模态框中画布区域不可见的问题。

## 根因分析
`.draft-modal` CSS 只设了 `max-height: min(92vh, 960px)` 而没有设 `height`。
由于 `.draft-modal` 是 flex 列布局容器，内部 `.draft-modal-body` 用 `flex:1` 期望填充剩余空间，
但没有确定的容器高度，flex-grow 无法分配空间。
`.draft-mm-canvas` 用 `height:100%` 引用父元素高度，结果解析为 0。
旧版 Markdown textarea 编辑器不受影响（因为 textarea 有固有尺寸），但思维导图画布子元素都是绝对定位，没有内容高度撑开容器。

## 所做改动

### CSS 修复（核心）
- **`styles.css`** `.draft-modal`: 将 `max-height: min(92vh, 960px)` 改为 `height: min(92vh, 960px)`，给 flex 容器一个确定的高度，使 `flex:1` 正确展开画布区域。

### JS 修复
- **`app.js`** `openDraftModal()`: 移除不必要的 `requestAnimationFrame` 包裹（`getBoundingClientRect` 触发同步重排，不需要延迟）。
- **`app.js`** `openDraftEditor()`: 将 `renderBuckets()` 移入 try-catch，避免异常时无反馈。

## 验证结果
- `npm test` ✅
- `app.js` 语法检查 ✅
- 服务端验证: CSS 文件正确返回 `height:min(92vh,960px)` ✅
