# 交付说明：Tab 创建子节点 + 连线方向修复

## 源任务
`msg-2026-04-08T18-06-17-816Z.md` — 思维导图选中节点按 Tab 无反应；连线需从 right 出 left 入

## 改动

### 1. Tab 创建子节点修复
- **根因**: 点击节点后，浏览器焦点从 canvas 转移到节点 div，导致 canvas 上的 keydown 事件不触发
- **修复**: 节点 mousedown 事件末尾追加 `draftMMCanvas.focus()`，确保选中节点后 canvas 重获焦点
- 子节点定位也从"下方偏右"调整为"右侧"（x+200, y + childCount*50），符合左到右思维导图布局

### 2. 连线方向（right → left）
- **改前**: 边从父节点底部中心出发，到子节点顶部中心（垂直走向）
- **改后**: 边从父节点右侧中心 `(x + pw, y + ph/2)` 出发，到子节点左侧中心 `(x, y + ph/2)`
- 控制点使用水平偏移 `dx = |x2-x1| * 0.45`，产生平滑的水平 S 曲线
