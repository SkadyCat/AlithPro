# Task: Tab should keep parent node selected

## Source task file
`msg-2026-04-09T02-34-45-456Z.md`

## Goal
When pressing Tab to create a child node, the **parent** node should remain selected instead of switching selection to the new child.

## Changes made
- **`application/public/mindmap.js`** — Tab key handler:
  - After `dmAddNode(..., false)` creates the child node, immediately re-select the parent node (`_dm.selectedIds.clear(); _dm.selectedIds.add(sel.id)`) and re-render.
  - Result: Tab creates the child node visually but keeps focus/selection on the parent, allowing rapid Tab presses to add multiple children.
