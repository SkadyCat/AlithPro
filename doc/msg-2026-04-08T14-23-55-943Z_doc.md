**Task goal**: 找到并修复界面突然"扯回去"的原因（视图重置/刷新）

## Root Cause

`loadWorkspace()` 每 6 秒被 polling 调用一次，它无条件调用 `window._mmSwitchWorkspace()`。  
`_mmSwitchWorkspace` 每次都会：
1. 重置 `panX = 0; panY = 0; scene.style.transform = ""`  
2. 调用 `load(); render()` 重建所有 DOM 节点

结果：每 6 秒思维导图视图被重置一次，导致"突然扯回去"。

## Fix (`application/public/app.js`)

添加 `lastStoreKey` 变量，在 `_mmSwitchWorkspace` 中判断工作区是否实际发生变化：

```js
// 只在工作区真正切换时才重置 pan + 重新渲染
const wsChanged = storeKey() !== lastStoreKey;
lastStoreKey = storeKey();
if (wsChanged) {
  panX = 0; panY = 0; scene.style.transform = "";
  selectedIds.clear(); load(); render();
}
refreshNodeDocContent(); // 每次都刷新文档内容
```

- **工作区未变（轮询）**：只调用 `refreshNodeDocContent()`，不重置视图
- **工作区实际切换**：完整重置 + 重渲染（正确行为）
