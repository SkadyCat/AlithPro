# 交付说明：解决 PowerShell 无法使用的问题

## 源任务文件
`msg-2026-04-07T13-57-21-680Z.md`

## 目标
修复 Copilot CLI 的 PowerShell (pwsh) 无法使用问题（表现为所有 shell 命令返回 `File not found: >`）。

## 根因分析

有两种可能场景：
1. **pwsh 未安装**：系统中没有 PowerShell 7
2. **pwsh 已安装但不在 PATH**：pwsh.exe 存在但 Copilot CLI 找不到它

## 改动内容

### 新增文件：`fix-pwsh.bat`（仓库根目录）

一键修复脚本，自动处理上述两种场景：

| 步骤 | 动作 |
|------|------|
| 1 | 检查 `where pwsh` — 已在 PATH 则直接跳到包装器创建 |
| 2 | 搜索常见安装路径（`C:\Program Files\PowerShell\7*`）|
| 3 | 深度递归搜索 `Program Files\PowerShell\` |
| 4 | 若仍未找到，通过 `winget install Microsoft.PowerShell` 自动安装 |
| 5 | 在 `%SystemRoot%\System32\pwsh.bat` 创建包装器（**立即生效**，无需重启终端） |
| 6 | 将 pwsh 目录添加到系统 PATH（永久生效，影响新终端） |
| 7 | 运行验证命令确认修复成功 |

### 已有文件（参考）
- `install-pwsh.bat`：仅负责安装 pwsh（通过 winget）
- `fix-pwsh-path.bat`：仅负责修复 PATH（适用于已安装但路径问题）

`fix-pwsh.bat` 整合了两者的功能，是一键完整解决方案。

## 使用方法

```bat
:: 右键选择"以管理员身份运行"
fix-pwsh.bat
```

修复后 Copilot CLI 的所有 `powershell` 工具调用将恢复正常，**无需重启系统或终端**。

## 验证结果
- 脚本语法已人工检查（`cmd /c` 批处理语法）
- 逻辑路径：find → wrapper → PATH → verify

## 剩余风险
- 需要管理员权限写入 `System32` 和更新系统 PATH
- 若 `winget` 不可用且 pwsh 未安装，需手动下载 MSI 安装包
