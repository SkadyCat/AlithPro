**Task goal**: Double-clicking a mind map node has no reaction

## Root Cause

The 📄 badge on root nodes had a `click` listener that opened the doc-viewer overlay on `click-1`. The overlay is `position:fixed; z-index:9999` covering the full viewport. So `mousedown-2` (the second click of the double-click) fired on the overlay, not the badge. Browsers fire `dblclick` on the **nearest common ancestor** of mousedown-1 and mousedown-2 targets — which was `document.body`, not the node element. The node's `dblclick` handler never fired.

## Fix (commit `0e77c1e`)

- **Removed** `badge.addEventListener("click", ...)` — badge is now a visual indicator only (no click handler)
- **Updated dblclick handler**: `if (n.linkedFile) openNodeDocViewer(n); else startEdit(n.id)`
  - Nodes with a linked file: double-click opens the 留言文档 / Agent文档 viewer
  - Nodes without a linked file: double-click opens rename dialog (unchanged behavior)
- Rename always accessible via right-click → ✏️ 重命名
