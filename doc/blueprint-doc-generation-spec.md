# 蓝图文档生成规范 v1.0

> 本规范定义 AlithEx 蓝图（思维导图）功能中"节点文档"的生成标准、命名规则和内容结构。

---

## 一、文档类型

| 类型 | 触发方式 | 保存位置 | 命名格式 |
|------|----------|----------|----------|
| **任务文档**（sessions 任务） | 子节点双击 → 输入内容 → Ctrl+Enter 提交 | `sessions/` | `msg-{ISO8601}.md` |
| **草稿文档**（留言草稿） | 留言弹窗 → 输入留言 → 提交 | `drafts/` | `draft-{ISO8601}.md` |
| **评论文档**（agent 回复） | Agent 处理节点消息后自动生成 | `doc/` | `comment-{ISO8601}_doc.md` |
| **交付文档**（任务完成） | Agent 完成任务后生成 | `doc/` | `{源文件名}_doc.md` |

---

## 二、任务文档格式（sessions/msg-*.md）

当子节点提交留言时，前端生成如下格式发送至 `POST /api/workspaces/{ws}/tasks`：

```markdown
# 思维导图留言

## 节点树概览

{从根节点到目标节点的完整缩进树，展示所有子树}

## 留言节点

- **节点ID**: {nodeId}
- **节点名称**: {node.text}
- **节点路径**: {root → parent → ... → 当前节点，用 → 连接}
- **时间**: {ISO 8601 UTC 时间戳}

## 留言内容

{用户输入的实际内容}
```

**规则：**
- 节点树概览显示全局树，而非仅当前节点的子树
- 有留言的节点标注 `[留言: N]`，有关联文档的节点标注 `[文档: 文件名]`
- "未命名"和"新节点"节点保持原样显示
- 留言节点路径用 ` → ` 连接各级祖先节点名

## 三、评论文档格式（doc/comment-*.md）

Agent 处理节点留言后，生成一份评论文档作为答复：

```markdown
# 节点文档：{节点名称}

**节点路径**：{完整路径}  
**创建时间**：{时间}  
**关联任务**：{msg 文件名，如适用}

## 需求摘要

{对节点留言内容的一句话摘要}

## 分析 / 方案

{Agent 的分析或实现方案}

## 实施步骤

1. ...
2. ...

## 验证标准

- {可观测的验收条件}

## 备注

{风险、阻塞、依赖项等}
```

---

## 四、交付文档格式（doc/{源文件名}_doc.md）

所有被处理过的 sessions 任务，完成后在 `doc/` 生成交付文档：

```markdown
# 交付说明

**源任务文件**：`{sessions 或 inprocess 中的文件名}`  
**任务目标**：{一句话描述}

## 所做改动

{每项改动的文件、内容和原因}

## 验证结果

{运行了哪些检查，结果如何}

## 剩余风险或阻塞项

{如无则写"无"}
```

## 五、节点与文档的绑定关系

```
node.linkedFile   —— 文件名（不含路径），如 msg-2026-04-08T12-35-43-468Z.md
node.linkedBucket —— 初始 bucket，如 "sessions"、"drafts"、"doc"
```

**查找顺序（多 bucket 降级）：**
1. `node.linkedBucket`（优先）
2. `processed/`
3. `inprocess/`
4. `sessions/`
5. `drafts/`

找到文件后更新 `node.linkedBucket` 缓存。

## 六、mind map 节点名约定

| 场景 | 节点文本 |
|------|---------|
| 根节点（新建） | "主题" |
| 子节点（默认） | "子节点" |
| 子节点（提交内容后） | 留言内容第一行，最多 50 字符 |
| 子节点（手动重命名） | 用户输入，F2 或右键触发 |

## 七、参考示例

以下节点树展示了包含留言和文档绑定的典型蓝图结构：

```
- 背包功能 [留言: 1] [文档: comment-...Z_doc.md]
  - 预制配装 [留言: 1] [文档: comment-...Z_doc.md]
    - lodout
      - 里面的所有控件(UInventorySlotWidgetBase) 的子类控件
        - 控件的EItemStatu计算逻辑
          - 重构ItemStatu的计算逻辑 [留言: 1] [文档: comment-...Z_doc.md]
            - 编辑模式下
              - DisableImg 默认隐藏 [留言: 1] [文档: comment-...Z_doc.md]
                - BUG:点击编辑按钮... [留言: 1] [文档: comment-...Z_doc.md]
```

每一个有 `[文档]` 标注的节点，对应 `doc/` 目录下的一个 `comment-*_doc.md` 文件，由 Agent 在处理该节点留言时自动生成。
