**Source task**: inprocess/msg-2026-04-08T13-41-02-162Z.md  
**Task goal**: Tab 生长出的子节点都没有内容 — 应自动打开内容输入对话框

## Root Cause

Tab → `addChild()` creates a node with `text: "子节点"` and no `linkedFile`. User had to manually dblclick the new node to open the bind dialog and type content. This extra step was invisible/unintuitive.

## Fix

In `addChild()`, after `render()`, immediately call `openNodeDocViewer(n)` on the new node. Since the node has no `linkedFile`, this opens the "✍️ 输入任务需求" bind dialog straight away — user can type content and submit without any extra double-click.

**Commit**: `0a5025b` — pushed to main

## UX After Fix

1. Select a node → press Tab
2. Bind dialog opens immediately with textarea focused
3. User types content → "🚀 提交并绑定" → node title + docContent set, session file created
4. Doc viewer opens showing the created task

## Risks

- If an overlay is already open when Tab is pressed, `openNodeDocViewer` silently skips (existing guard). Node is still created; user can dblclick to open dialog manually.
