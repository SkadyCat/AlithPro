**Task goal**: Multi-select delete must delete ALL selected nodes, not just the last one

## Root Cause

The Delete/Backspace keydown handler was on the **canvas element** (which needs explicit focus).
After Ctrl+clicking .mm-node elements, the canvas is not guaranteed to have focus, so the
keyboard event never reached the handler. Only the "last" right-clicked node (held in selectedIds
after any accidental plain-click) would get deleted via context menu.

## Changes Made

**app.js** (commit `a14e734`):
- Removed Delete/Backspace from the `canvas.addEventListener("keydown", ...)` handler
- Added a new `document.addEventListener("keydown", ...)` handler that:
  - Skips when `editingId != null` (rename dialog is open)
  - Skips when active element is INPUT or TEXTAREA
  - Calls `deleteSelected()` for Delete/Backspace when `selectedIds.size > 0`
- `deleteSelected()` already correctly iterates all nodes in `selectedIds` (Set)
