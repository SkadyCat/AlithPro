# 交付说明：Copilot CLI Shell 工具 "File not found" 修复

**源任务文件：** `msg-2026-04-07T13-20-44-060Z.md`  
**任务目标：** 分析另一台设备上 Copilot Agent 所有 shell 命令失败（"File not found"）的原因，并提供一键解决方案

---

## 根本原因

**Copilot CLI 的 shell 工具依赖 `pwsh`（PowerShell 7）**。

当设备上没有安装 PowerShell 7 时，Copilot CLI 尝试调用 `pwsh.exe` 执行 shell 命令，但找不到该可执行文件，因此所有命令（包括最基础的 `Get-Date`、`echo`、`hostname`）都以 `File not found` 报错退出。

设备上即使有 Windows 自带的 `powershell.exe`（5.x 版本），也不足够——Copilot CLI 明确依赖 `pwsh`。

### 诊断依据

从任务文件中的运行日志可见：
- `powershell.exe -Command "Write-Host 'test'"` → File not found  
- `cmd /c echo test` → File not found  
- `Get-Date` → File not found  
- `python -c "print('hello')"` → File not found  

所有命令均失败，表明底层 shell 可执行文件无法定位，而非脚本逻辑错误。

---

## 解决方案

### 一键修复

在根目录提供了修复脚本：

```
install-pwsh.bat
```

以**管理员身份**双击运行，脚本会：
1. 检测 `pwsh` 是否已安装
2. 若未安装，自动通过 `winget` 安装 PowerShell 7（`Microsoft.PowerShell`）
3. `winget` 不可用时，提示手动下载地址

### 手动安装（备用）

```bat
winget install --id Microsoft.PowerShell --source winget --accept-package-agreements
```

或从以下地址下载 `.msi` 安装包：

> https://github.com/PowerShell/PowerShell/releases/latest

安装时勾选 **"Add PowerShell to Path Environment Variable"**。

### 安装后

重启终端（或 VSCode/Cursor）后重新启动 Copilot CLI，所有 shell 工具即可正常工作。

---

## 验证方法

安装后运行：

```powershell
pwsh --version
# 预期输出类似：PowerShell 7.6.0
```

Copilot CLI 中任意 shell 命令不再报 "File not found"。

## 剩余风险 / 阻塞项

- 若目标设备处于企业内网且无法访问 winget / GitHub，需要管理员手动从内网源安装 PowerShell 7
- Windows 7 / Server 2008 R2 不支持 PowerShell 7，需要升级操作系统
