## 根因

`startPolling()` 每 6 秒调用一次 `loadWorkspace(state.currentWorkspace)`，触发链：

```
loadWorkspace → _mmSwitchWorkspace → load() → nodes = JSON.parse(localStorage)
```

`load()` 用 localStorage 数据**完全替换 `nodes` 数组**（创建全新的对象）。

旧代码中 `showRenameDialog(n)` 接收的 `n` 是旧 `nodes[]` 里的引用。当 confirm() 执行时：
1. `n.text = "新名字"` — 更新旧对象（已不在新 `nodes[]` 里）
2. polling 触发 `load()` → `nodes` 被替换为新数组（含 `text=""`）
3. `save()` 保存新 `nodes`（text="" ）到 localStorage
4. `render()` → 显示"节点"（`n.text || "节点"` 的 fallback）

## 修复

**`showRenameDialog(nodeId)`** — 改为接受 id 而非引用：
- `confirm()` 里 `target = nodes.find(nd => nd.id === nodeId)` — 在**当前活跃的** `nodes[]` 里重新查找，确保操作的是 polling reload 后的新对象
