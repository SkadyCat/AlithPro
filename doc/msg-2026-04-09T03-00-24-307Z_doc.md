# Task: Fix mind map pan position not persisting across reopens

## Source task file
`msg-2026-04-09T03-00-24-307Z.md`

## Goal
The mind map canvas pan position should be remembered between editor open/close cycles instead of always resetting to center on root.

## Changes made
- **`application/public/app.js`** — `openDraftModal()`:
  - Previously: always forced `panX`/`panY` to center on root node, overriding saved values from `dmParse()`.
  - Now: only centers on root when `panX === 0 && panY === 0` (default/no saved pan), otherwise respects the saved pan position.

## Verification
- `node --check app.js` — syntax OK.
- `dmSerialize()` already includes `panX`/`panY` in saved data, so after manual save the position persists.

## Remaining risks / blockers
- Pan position is only saved when the user clicks the save button. Auto-save after pan could be a future enhancement.
