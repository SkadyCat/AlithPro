# 交付说明：修复 pwsh 已安装但 Copilot CLI 仍报错的问题

**源任务文件：** `msg-2026-04-07T13-27-14-083Z.md`  
**任务目标：** 修复 pwsh 7.5.4 已安装，但 Copilot CLI 仍然无法执行 shell 命令的问题

---

## 根本原因（深层）

pwsh 是安装后才写入 PATH 的，**已经打开的终端/Copilot CLI 进程在安装前就已启动，不会自动继承新增的 PATH 条目**。因此：

- `where pwsh` 在新终端中可以找到
- 但运行中的 Copilot CLI 进程的 `PATH` 环境变量快照里没有 pwsh 目录
- 导致所有 shell 工具调用依然以 "File not found" 失败

---

## 解决方案

### 新增：`fix-pwsh-path.bat`（无需重启终端）

**以管理员身份运行 `fix-pwsh-path.bat`**，脚本执行：

1. **自动定位** `pwsh.exe`（扫描标准安装路径）
2. **在 `%SystemRoot%\System32` 创建 `pwsh.bat` 包装器** → 当前所有已运行进程（包括 Copilot CLI）立即可通过 `pwsh` 调用，无需重启
3. **更新系统 PATH**（via `setx /M`）→ 后续新开终端永久生效
4. **自验证**：调用 pwsh 确认修复成功

### 根本性预防措施（无脚本需求）

若能重启 Copilot CLI（不用重启系统）：关闭当前终端窗口，新开终端后重新启动 Copilot CLI。pwsh 已在 PATH 中，直接生效。

---

## 改动摘要

| 文件 | 操作 |
|------|------|
| `fix-pwsh-path.bat` | 新建 — 无需重启的 pwsh PATH 修复脚本 |

---

## 剩余风险 / 阻塞项

- 必须**以管理员身份**运行，才能写入 `System32` 和更新系统 PATH
- 若企业策略禁止在 `System32` 创建文件，可改为将 pwsh 目录加入用户 PATH 并重启终端
