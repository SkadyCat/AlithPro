**源任务文件**: msg-2026-04-08T06-49-10-352Z.md  
**任务目标**: 卡片右键新增"创建子任务"功能 + processed 列按根节点分组

## 所做改动

### server.js
- 新增 parseSubtaskParent(content) — 从文件内容首行解析父任务文件名
- 新增 parseSubtaskChain(content) — 解析任务链路行，用于构建子任务链路
- listCards() 返回值新增 subtaskParent 字段（所有 bucket）
- 新增路由 POST /api/workspaces/:workspace/tasks/subtask：
  - 接受 { parentBucket, parentFileName, content }
  - 读取父任务文件，提取已有链路，追加父文件名后写入 sessions/
  - 新文件格式：这个文本是{父文件名}的迭代子任务，它的需求是：\n{内容}\n\n---\n任务链路：{完整链路}

### public/index.html
- 新增 #subtask_modal 模态框（含文本域、确认/取消按钮、父任务元信息显示）

### public/styles.css
- 新增 .subtask-modal* 样式（与 draft-modal 风格一致）
- 新增 .processed-group-header 分组标题样式

### public/app.js
- 新增 DOM refs：subtaskModal、subtaskModalMeta、subtaskModalInput、subtaskModalClose、subtaskModalSubmit
- 新增函数：openSubtaskModal(bucket, fileName)、closeSubtaskModal()、submitSubtask()
- uildCard() contextmenu 菜单新增"创建子任务"（隐藏于 drafts/workflows）
- enderBuckets() processed 列改为调用 enderProcessedGrouped()
- 新增 esolveRoot(fileName, parentMap) — 递归找根节点（含循环防护）
- 新增 enderProcessedGrouped(container, items) — 按根节点分组渲染，有子任务的组显示 📁 分组标题
- 新增 subtaskModal 事件监听：点击背景关闭、Escape 关闭、Ctrl+Enter 提交
