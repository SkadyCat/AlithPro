## 根本原因

服务器（PID 14464）启动于代码变更之前（22:44），运行旧版 server.js（无 /tasks/subtask 路由）。  
浏览器可能缓存了旧版 index.html，导致 subtask_modal DOM 元素不存在，openSubtaskModal 因 null 检查静默返回。

## 所做改动

### public/app.js
- 将 openSubtaskModal 改为先调用 ensureSubtaskModalDOM() 动态创建 modal（如 DOM 中不存在则注入）
- 所有 modal 操作改用 getSubtaskEl(id) 按需查找，不依赖加载时获取的 const 引用
- 事件监听在 ensureSubtaskModalDOM() 中注入，静态 DOM refs 的监听保留作为兼容后备
- 结果：无论 index.html 版本新旧，点击菜单项都能正常弹出 modal

### 服务器重启
- Stop-Process -Id 14464（旧进程）
- 启动新 node server.js 进程（PID 25328），新路由 POST /tasks/subtask 已生效

## 验证结果
