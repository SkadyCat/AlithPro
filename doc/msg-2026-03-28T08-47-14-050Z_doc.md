# 交付说明

- 源任务文件：$taskName
- 任务目标：修正 http://localhost:7439/dev 页面读取的任务目录，使其使用根级 Alith\sessions，而不是误读 Alith\application 下的目录。

## 所做改动

1. 更新 pplication\server.js：
   - 将应用目录与仓库根目录拆分为独立概念。
   - 新增根级工作区识别逻辑，把当前仓库根 Alith 暴露为真实 workspace。
   - 让 /api/workspaces、/api/workspaces/:workspace 等接口在请求 Alith 时读取根级 sessions、inprocess、processed、doc、logs。
2. 更新 pplication\launch-agent.py：
   - 启动 Agent 时按同一规则解析 workspace。
   - 当 workspace 为 Alith 时，优先使用根级 Alith\run.bat，避免继续落到 pplication\workspace\Alith。

## 验证结果

- cd application && npm test：通过。
- python -m py_compile application\launch-agent.py：通过。
- 启动本地服务后，请求 http://localhost:7439/api/workspaces：返回包含 Alith,test，确认 Alith 已被识别。
- 请求 http://localhost:7439/api/workspaces/Alith：返回根级队列统计（sessions=0, inprocess=1, processed=4），确认 /dev 已读到根级 Alith 工作区。
