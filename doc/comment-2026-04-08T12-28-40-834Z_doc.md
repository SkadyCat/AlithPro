# 节点文档：BUG — 点击编辑按钮未刷新 DisableImg

**节点路径**：背包功能 → 预制配装 → lodout → 里面的所有控件(UInventorySlotWidgetBase) 的子类控件 → 控件的EItemStatu计算逻辑 → 重构ItemStatu的计算逻辑 → 编辑模式下 → DisableImg 默认隐藏 → BUG:点击编辑按钮，需要刷新一遍所有控件的 DisableImg 为隐藏，但是现在点进去，还是显示的。  
**创建时间**：2026-04-08T12:28:40.834Z  
**关联任务**：msg-2026-04-08T13-00-15-208Z.md

---

## 需求摘要

点击编辑按钮进入编辑模式时，所有 `UInventorySlotWidgetBase` 子类控件的 `DisableImg` 应立即刷新为隐藏状态，但当前存在残留显示。

---

## 分析

### 根本原因

1. **对象池残留**：UMG 控件来自对象池，复用时不会自动重置所有子 Widget 状态，`DisableImg` 的 `Visibility` 停留在上一次的值。
2. **编辑模式切换时未主动刷新**：进入编辑模式的逻辑（通常在 `OnEditModeEnter` 或 `OnClickEditButton`）仅更新了数据层 / EItemStatu，未对所有当前可见的插槽控件执行一次 DisableImg 的状态重置。
3. **EItemStatu 计算顺序问题**：在编辑模式下 `ItemStatu` 应排除 `NoInStash` / `NoOffer` 等非编辑态枚举，从而触发 `DisableImg` 隐藏，但如果 `UpdateWidgetState` 未被主动调用，控件就不会响应模式变化。

---

## 修复方案

### 方案 A：编辑模式入口统一刷新（推荐）

在点击"编辑按钮"的回调里（`OnEditButtonClicked` / `EnterEditMode`），遍历所有已激活的 `UInventorySlotWidgetBase` 子类控件并调用 `SetDisableImgVisibility(ESlateVisibility::Hidden)`：

```cpp
void UInventoryWidget::EnterEditMode()
{
    bEditMode = true;

    // 遍历所有插槽控件，重置 DisableImg
    for (UInventorySlotWidgetBase* Slot : GetAllSlotWidgets())
    {
        if (Slot && Slot->DisableImg)
        {
            Slot->DisableImg->SetVisibility(ESlateVisibility::Hidden);
        }
    }

    // 再触发正常的状态更新流程
    RefreshAllSlotStates();
}
```

### 方案 B：对象池激活时重置（根治）

在对象池控件的 `NativeOnListItemObjectSet` / `OnReset` / `Activate` 回调里强制重置所有可能残留的状态：

```cpp
void UInventorySlotWidgetBase::ResetWidgetState()
{
    if (DisableImg) DisableImg->SetVisibility(ESlateVisibility::Hidden);
    // ... 重置其他残留状态
}
```

### 方案 C：在 `UpdateItemStatu` 末尾加编辑模式覆盖

```cpp
void UInventorySlotWidgetBase::UpdateItemStatu(EItemStatu NewStatu)
{
    // ... 正常的 statu 更新逻辑 ...

    // 编辑模式强制隐藏 DisableImg
    if (bEditMode && DisableImg)
    {
        DisableImg->SetVisibility(ESlateVisibility::Hidden);
    }
}
```

## 实施步骤

1. 确认"编辑按钮"的 BP/C++ 回调函数名，添加遍历刷新逻辑（方案 A）。
2. 同时在对象池的重置回调里补充 `DisableImg->SetVisibility(Hidden)` 防御性重置（方案 B）。
3. 在 `UpdateItemStatu` 末尾加编辑模式保护（方案 C，防止后续逻辑又写回 Visible）。
4. 进入编辑模式后点击 Loadout，验证所有插槽 DisableImg 为隐藏。
5. 切换到非编辑模式后，NoInStash 插槽应恢复 DisableImg 显示。
