# 交付说明

- 源任务文件：$taskName
- 任务目标：修复 pplication\workspace\UIEditor\run.bat 启动的 agent 错扫到 Alith\sessions 而不是 UIEditor\sessions 的问题。

## 根因

问题不是 UIEditor\wait-for-session-doc.bat 路径写错；它本身就是扫本地 UIEditor\sessions。

真正的根因是：当前 Alith 根目录已经被初始化成 Git 仓库，因此从 UIEditor 子目录执行 Git 根解析时，返回的 repo root 变成了上层 Alith：

- git -C application\workspace\UIEditor rev-parse --show-toplevel
- 实际结果：$gitRoot

这会让 Copilot CLI 在某些启动场景下优先采用父级 Alith 根仓库的说明文件与工作区规则，从而把扫描逻辑导回 Alith\sessions。

## 所做改动

1. 确认 pplication\workspace\UIEditor\run.bat 已固定使用 cd /d %~dp0，保证 launcher 先切到自身目录。
2. 更新根级 \.github\copilot-instructions.md：新增“如果当前工作目录已经位于 pplication\workspace\{name} 且本地自带 sessions/、inprocess/、processed/、doc/、logs/，则直接把当前目录视为活动工作区根目录”的规则。
3. 同步更新以下现有子工作区说明文件，确保它们在被单独启动时也明确采用本地目录，不回退到父级仓库：
   - pplication\workspace\UIEditor\.github\copilot-instructions.md
   - pplication\workspace\test\.github\copilot-instructions.md
   - pplication\workspace\MagicWorld\.github\copilot-instructions.md

## 验证结果

- UIEditor 本地目录确实存在自己的 sessions\、inprocess\、processed\、doc\、logs\。
- UIEditor\run.bat 当前已包含 cd /d %~dp0。
- Git 根解析结果证明当前启动上下文会命中父级 Alith 仓库，因此上述说明文件修正是必要的。
- 本次改动主要修正规则与启动约束；由于问题涉及外部 Copilot CLI 会话启动行为，未在当前会话内直接启动一个新的真实 UIEditor Copilot 会话做端到端回归，但根因与修复点已经闭合。

## 剩余风险或阻塞项

- 若已有旧的 UIEditor Agent 会话仍在运行，它不会自动加载更新后的说明文件；需要重新启动该 UIEditor Agent 会话。
- 无其它阻塞项。
