# 交付说明

**源任务文件**: `msg-2026-04-06T05-51-19-807Z.md`

## 任务目标

right_info 侧边栏中卡片内容和文档内容显示区域太小，阅读不便。要求：点击卡片后以弹窗（modal）形式展示完整内容。

## 改动摘要

### `application/public/index.html`
- 新增 `#content_modal` 弹窗 DOM 结构，放置在 `#workflow_modal` 之前。
- 包含：标题栏（eyebrow + 文件名 + 路径元数据 + 关闭按钮）、可滚动内容区域 `#content_modal_body`。

### `application/public/styles.css`
- 新增 `.content-modal`、`.content-modal-header`、`.content-modal-title-wrap`、`.content-modal-meta`、`.content-modal-body`、`.content-modal-block`、`.content-modal-block-label`、`.content-modal-pre` 等样式。
- 弹窗宽度 `min(860px, 100%)`，最大高度 88vh，风格与现有 `.draft-modal` 保持一致（深色毛玻璃背景、圆角、边框发光）。
- 内容区使用等宽字体、行高 1.75，保证长文本可读性。

### `application/public/app.js`
- 新增 `contentModal`、`contentModalClose`、`contentModalTitle`、`contentModalEyebrow`、`contentModalMeta`、`contentModalBody` DOM 引用。
- 新增 `showContentModal(detail)` 函数：填充标题、路径、主内容及可选关联文档块，显示弹窗并锁定 body 滚动。
- 新增 `closeContentModal()` 函数：隐藏弹窗，恢复 body 滚动。
- 修改 `loadItem()`：在 `renderDetail()` 和 `renderBuckets()` 之后调用 `showContentModal(detail)`，使每次点击卡片都弹出内容弹窗。
- 新增事件监听：点击遮罩层关闭、点击"关闭"按钮关闭、按 Escape 键关闭弹窗。

## 验证结果

- `node --check public/app.js` 通过，无语法错误。
- HTML 结构与现有 draft_modal / workflow_modal 模式一致。
- 关闭行为：三种方式（关闭按钮、遮罩点击、Escape 键）均已实现。
