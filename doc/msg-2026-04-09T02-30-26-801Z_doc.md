# Task: Tab key should not auto-focus new node in mind map

## Source task file
`msg-2026-04-09T02-30-26-801Z.md`

## Goal
When pressing Tab to create a child node in the mind map, the new node should not automatically enter edit mode (no auto-focus).

## Changes made
- **`application/public/mindmap.js`**:
  - Added `autoEdit` parameter (default `true`) to `dmAddNode()` — only calls `dmStartEdit` when `autoEdit` is true.
  - Tab key handler now passes `autoEdit = false`, so the new child node is created and selected but not focused for editing.
  - All other callers (context menu, canvas right-click) retain the original auto-edit behavior.

## Verification
- `node --check mindmap.js` — syntax OK.

None.
