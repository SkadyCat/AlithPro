# 交付说明

- **源任务文件**: `msg-2026-04-05T07-47-27-080Z.md`
- **任务目标**: 修复“新创建的 workspace 点击启动 Agent 后，旧版坏 `run.bat` 仍被执行并报中文乱码命令错误”的问题。

## 所做改动

1. 检查 `application\launch-agent.py` 与 `application\server.js` 的启动链，确认“启动 Agent”最终由 `launch-agent.py` 解析并执行 workspace 下的 `run.bat`。
2. 判断出问题根因并不只是 workspace 创建逻辑，还包括：
   - 旧的 Node 服务实例仍可能生成旧版坏 `run.bat`；
   - 即使磁盘上代码已修复，`launch-agent.py` 在启动前如果不主动修正 launcher，仍可能执行遗留坏文件。
3. 为 `application\launch-agent.py` 增加 launcher 自愈逻辑：
   - 新增读取现有 `run.bat` 模型配置的能力；
   - 新增 `build_run_bat_content()` / `build_export_log_bat_content()`；
   - 新增 `write_text_if_changed()`；
   - 新增 `normalize_workspace_launchers()`，在每次真正启动前先把 workspace 的 `run.bat` / `export_log.bat` 规范化为 **ASCII + CRLF**。
4. 让 `launch-agent.py` 在 `main()` 中先执行 launcher 自愈，再解析并运行批处理文件。

## 验证结果

1. 执行 `application` 工作区现有检查命令：`npm test`
2. `node --check server.js` 通过。
3. 构造一个带有旧版 UTF-8 + LF 中文注释 `run.bat` 的临时 workspace，直接调用 `launch-agent.py` 中的自愈逻辑，确认它会把 launcher 重写为标准格式。
4. 对自愈后的临时 workspace 直接通过 `cmd.exe` 执行 `run.bat`，启动时不再出现：
   - `嗗惎鍔ㄨ剼鏈? 不是内部或外部命令`
   - `杩欓噷淇敼绔彛 不是内部或外部命令`
   - `剧疆浠ｇ悊 不是内部或外部命令`

## 剩余风险或阻塞项

- 当前未发现阻塞项。
- 如果有更早期的外部脚本绕过 `launch-agent.py` 直接执行历史坏 `run.bat`，仍需先触发一次自愈或手动覆盖；但“启动 Agent”这条官方路径已经被修复。
