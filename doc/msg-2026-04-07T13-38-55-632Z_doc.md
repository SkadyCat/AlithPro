# 交付说明：修复 bat 脚本乱码问题

**源任务文件：** `msg-2026-04-07T13-38-55-632Z.md`  
**任务目标：** 修复 bat 脚本乱码，并确认脚本运行状态

---

## 问题分析

**乱码原因：** bat 文件以 UTF-8 编码保存，但 Windows cmd.exe 默认使用 GBK (CP936) 编码，
中文字符在 cmd 中显示为乱码（如 `瀹夎` 等）。

**脚本运行状态：** `install-pwsh.bat` 正常运行完成。它检测到 pwsh 7.5.4 已安装，
输出了"已安装"分支后退出。这是正确行为——脚本没有报错，只是提示 pwsh 已存在。

---

## 修复内容

将 `install-pwsh.bat` 和 `fix-pwsh-path.bat` 全部重写为**纯 ASCII 英文**：
- 0 个非 ASCII 字节，任何 Windows 代码页下均不会乱码
- 内容完整保留所有功能，仅将提示文字改为英文

---

## 下一步操作

pwsh 已安装但 Copilot CLI 仍报错，需要运行：

```
fix-pwsh-path.bat  （以管理员身份运行）
```

成功后输出：
```
[OK] Found pwsh: C:\Program Files\PowerShell\7.5.4\pwsh.exe
[OK] Wrapper created: C:\Windows\System32\pwsh.bat
[OK] ... already in System PATH  -or-  [OK] System PATH updated
[PASS] pwsh version: 7.5.4
[SUCCESS] Fix complete!
```

之后无需重启，Copilot CLI 立即可用。
