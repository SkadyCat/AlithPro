## Basic Architecture Analysis

- `run.bat` is the minimal local entrypoint that launches the Copilot CLI for this service.
- `export_log.bat` is a diagnostics-oriented launcher that runs Copilot with verbose logging redirected to `logs\copilot-shell.log`.
- `.github\hooks\autonomous-session.json` injects the autonomous startup prompt so a fresh session immediately scans the task queue.
- `.github\scripts\launch-agent.ps1` is the orchestration wrapper for a named workspace: it creates the workspace log directory, launches the batch loop, streams stdout and stderr to the console, and mirrors cleaned output into `agent-realtime.log`.
- `.github\scripts\agent-watchdog.ps1` is the safety process: it monitors `agent-realtime.log` activity and kills the agent tree if the workspace appears stalled.
- `.github\scripts\wait-for-session-doc.ps1` is the event-driven watcher used for queue waiting. It resolves the active workspace, watches the `sessions\` folder for new Markdown files, and emits heartbeat log entries during continuous waiting.
- The repository itself is organized around a task lifecycle: `sessions\` holds queued task documents, `inprocess\` holds the active task, `processed\` stores completed task documents, `doc\` stores delivery/session notes, and `logs\` stores operational logs.
