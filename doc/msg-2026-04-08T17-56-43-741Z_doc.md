# 交付说明：修复思维导图编辑器在 dev 页面不可用

## 源任务
`msg-2026-04-08T17-56-43-741Z.md` — 用户反馈点击思维导图卡片弹出的不是思维导图编辑器

## 任务目标
修复思维导图编辑器在 `/dev` 页面无法正常显示的问题。

## 根因分析
`dev.html` 仍保留旧版草稿编辑器（Markdown textarea + 预览），未同步 `index.html` 的思维导图画布改动。
`app.js` 中的 `openDraftModal()` 依赖 `draft_mm_canvas`/`draft_mm_scene`/`draft_mm_svg` 等 DOM 元素，
在 `dev.html` 中这些元素不存在，导致模态框打开后显示空白的旧版 Markdown 编辑器。

## 所做改动

### 1. dev.html — 同步思维导图 UI (主修复)
- 替换旧的 textarea/preview 草稿模态框为思维导图画布（`draft_mm_canvas` + `draft_mm_scene` + `draft_mm_svg`）
- 添加新建思维导图对话框（`new_mm_modal`）
- 模态框标题从 "草稿编辑器" 改为 "思维导图"
- 添加操作提示条 `draft_mm_hint`
- 按钮文案从 "保存草稿" 改为 "保存"
- Bucket 标题 "草稿" → "思维导图"
- 新建按钮 "新建草稿" → "新建导图"
- 统计标签 "草稿" → "导图"

### 2. app.js — 修复节点事件处理器跨草稿污染
- `dmRender()` 中节点的 mousedown/dblclick/contextmenu 事件处理器改为通过 `_dm.nodes.find()` 动态解析当前节点，避免切换草稿后闭包引用旧节点对象。

### 3. app.js — 之前已修复的问题（Task 8）
- compact 卡片模式补充 `isMindMapJSON` 检查隐藏 JSON 预览
- `openDraftModal()` 画布居中逻辑包裹在 `requestAnimationFrame()` 中

## 验证结果
- `npm test` ✅
- `app.js` 语法检查 ✅
- dev.html 验证: 所有思维导图 DOM 元素就绪，旧编辑器元素已移除 ✅
- 服务端验证: `/dev` 页面正确返回更新后的 HTML ✅

## 风险 / 备注
- 用户浏览器可能缓存旧版文件，需 Ctrl+Shift+R 强制刷新
