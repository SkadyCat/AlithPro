**Task goal**: Fix rename failure

## Root Cause

Dialog opened from a context menu item's `mousedown` handler. After the menu item was removed from DOM via `hideContextMenu()`, the pending `mouseup`+`click` events fired on the newly-appended `mm-msg-overlay` backdrop. The backdrop had a `click` listener that dismissed the dialog when `e.target === overlay`, so the dialog opened and immediately closed.

## Fix

Wrapped the backdrop `click` listener in `setTimeout(200ms)` in both `showRenameDialog` and `showMsgDialog`, so it doesn't activate during the same event cycle that opened the dialog. The Cancel button and Escape key still work immediately.
