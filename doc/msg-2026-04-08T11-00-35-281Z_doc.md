### 数据结构

每个节点（node）：
`json
{
  "id": 1,           // 唯一 ID（递增）
  "x": 120,          // 画布像素 X 坐标
  "y": 80,           // 画布像素 Y 坐标
  "text": "根节点",   // 节点文字
  "parentId": null,  // 父节点 ID（null = 根节点）
  "linkedFile": "draft-xxx.md",   // 关联的 Alith 文档文件名（根节点专用）
  "linkedBucket": "drafts"        // 文档所在 bucket
}
`

持久化：`localStorage["alith:mindmap:{workspaceName}"]`，格式 `{ nodes, nextId }`。  
切换工作区时自动 load + render（通过 `window._mmSwitchWorkspace` 钩子）。

### 核心操作

| 操作 | 触发方式 |
|------|---------|
| 放置根节点 | 画布右键 → 菜单「📍 放置根节点」|
| 添加子节点 | 节点右键 → 「➕ 添加子节点」或选中节点按 Tab |
| 重命名节点 | 节点右键 → 「✏️ 重命名」或双击节点 → 弹窗输入 |
| 留言 | 节点右键 → 「💬 留言」→ 弹窗输入 |
| 删除节点 | 节点右键 → 「🗑️ 删除节点」或选中后按 Delete/Backspace（递归删除子树）|
| 拖动节点 | 按住节点拖拽 |
| 清空画布 | 工具栏「清空」按钮（确认弹窗）|

### 任务系统集成

**根节点命名后自动创建 Draft 文档**  
`syncRootNodeToTask(n)`：
- 首次命名 → `POST /api/workspaces/{ws}/drafts` 创建 draft，存入 `linkedFile` / `linkedBucket`
- 再次重命名 → `PUT` 更新 draft 内容 + 更新 `setChainAlias`

**留言 = 子链路文档**  
`addMessageToNode(n)`：
- 向上追溯到链路根节点，找到其 `linkedFile`
- `POST /api/workspaces/{ws}/tasks/subtask`，创建以根文档为父的子任务
- 提交后刷新主面板 bucket 列表

---

### 渲染机制

- 节点：DOM `div.mm-node`，绝对定位在 `#mindmap_canvas`
- 连线：SVG `path.mm-edge`（贝塞尔曲线），overlay 在画布之上但 `pointer-events:none`
- 根节点有 `linkedFile` 时显示 📄 徽章
- 每次操作后全量重绘（remove all + recreate）

---

### 已知限制

1. 全量重绘：每次操作都销毁重建所有 DOM 节点，大量节点时可能有性能问题
2. 无撤销/重做功能
3. 节点坐标固定在画布左上角为原点，不支持画布平移/缩放
4. 链路文档创建于 drafts bucket，不会自动进入 sessions 队列（agent 不会自动处理）
5. 留言仅支持文字，无法附件或格式化
