# 交付说明

- **源任务文件**: `msg-2026-04-05T07-30-52-678Z.md`
- **任务目标**: 修复创建 workspace 时默认生成异常 `run.bat` 的问题，避免新 workspace 再次带出无法正常启动的批处理脚本。

## 所做改动

1. 定位到 `application\server.js` 中的 workspace 启动器生成逻辑：
   - `buildRunBatContent(model)`
   - `buildExportLogBatContent(model)`
   - `syncWorkspaceLaunchers(workspaceDir, preferredModel)`
2. 将批处理模板统一改为 **CRLF** 行结尾，避免 `cmd.exe` 把命令首字符误读，导致 `copilot` 被解析成 `pilot`。
3. 将 `run.bat` 模板中的标题和注释改为 ASCII 文本，避免新创建 workspace 的批处理文件再受编码兼容性影响。
4. 同步把 `export_log.bat` 模板也改为 CRLF，避免同类问题在另一个批处理入口重复出现。

## 验证结果

1. 执行 `application` 工作区现有检查命令：`npm test`
2. `node --check server.js` 通过。
3. 复查源码，确认生成器现在输出的是 CRLF 批处理内容，且 `run.bat` 模板使用 ASCII 文本。

## 剩余风险或阻塞项

- 当前未发现阻塞项。
- 旧 workspace 里已经生成过的异常 `run.bat` 不会被这次源代码修改自动回写；后续需重新生成或手动覆盖这些旧文件，新的 workspace 才会直接受益。
