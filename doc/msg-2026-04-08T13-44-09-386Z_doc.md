## Root Cause

`bindNodeToTask` set `docContent = lines.slice(1).join("\n")` — only lines after the first. Single-line messages produced `docContent = null`, so the card body stayed empty. User typed a message and the card showed nothing in the preview area.

## Fix

- `bindNodeToTask`: `fresh.docContent = content.trim().slice(0, 300)` — full submitted content, always
- `refreshNodeDocContent` (lazy-load for existing nodes): same — `n.docContent = raw.trim().slice(0, 300)`

Title (`mm-node-title`) still shows `firstLine` as a bold summary. Preview (`mm-node-preview`) now shows the full message content (clamped to 5 lines by CSS `-webkit-line-clamp`).

**Commit**: `438c604` — pushed to main

After Tab → type message → submit:

- Card body = full message text (up to 300 chars, CSS clamps display to 5 lines)
- Existing bound nodes without docContent are lazily back-filled on next page load
