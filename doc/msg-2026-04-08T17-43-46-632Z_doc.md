# 交付说明：修复思维导图卡片显示和编辑功能

## 源任务
`msg-2026-04-08T17-43-46-632Z.md` — 用户反馈思维导图功能不可用

## 任务目标
修复两个问题：1) 卡片仍显示原始 JSON 而非仅显示名称；2) 点击卡片后思维导图编辑器不能正常显示。

## 所做改动

### Bug 1：Compact 卡片模式 JSON 预览未隐藏
- **文件**: `application/public/app.js` `buildCard()` compact 分支
- **根因**: 草稿卡片始终以 compact 模式渲染（通过 `renderChainTree`），但 compact 分支没有像 non-compact 分支那样检查并隐藏思维导图 JSON 预览。
- **修复**: 在 compact 分支的 `treePreview` 赋值后添加 `isMindMapJSON` 检查，当 `item.preview` 以 `'{"nodes"'` 开头时将 `treePreview` 置空。

### Bug 2：模态框打开后画布尺寸为 0 导致节点不可见
- **文件**: `application/public/app.js` `openDraftModal()`
- **根因**: `openDraftModal` 先显示模态框（`hidden = false`），然后立即调用 `getBoundingClientRect()` 获取画布尺寸。但浏览器尚未完成布局重排，画布的 width/height 为 0。计算出的 panX/panY 为负数（约 -370, -274），把所有节点推出可视区域。
- **修复**: 将画布居中和初始渲染逻辑包裹在 `requestAnimationFrame()` 中，确保浏览器完成布局后再读取画布尺寸。
