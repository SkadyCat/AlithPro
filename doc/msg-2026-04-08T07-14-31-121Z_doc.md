## 根因（多点）

1. **Toast z-index 被遮挡**：.toast 的 z-index: 9999，而 .modal-backdrop 的 z-index: 10001。当 API 调用失败（或服务器未重启仍返回 404）时，错误 toast 被模态框遮挡，用户看不到任何反馈，误以为"没反应"。

2. **重复事件监听器**：app.js 底部同时存在两组 subtask modal 事件绑定代码（一组使用 if-guard，另一组使用可选链）。虽然不导致点不了，但会造成 submitSubtask 被调用两次，可能引发双重请求。

3. **静默返回无提示**：submitSubtask 的守卫条件 if (!_subtaskContext || !state.currentWorkspace) return; 静默返回，用户无任何提示。
