## Root Cause

After the pan refactor (commit `3282610`), `.mindmap-scene` was added as a wrapper with `position:absolute; top:0; left:0` but **no explicit width/height**. The SVG inside used `width:100%; height:100%` which resolved to **0×0**, making all `<path>` edge elements invisible (drawn in a zero-size viewport).

## Fix

Set `.mindmap-scene { width:8000px; height:8000px }` — a large fixed "infinite canvas" area. The SVG fills this space (`width:100%; height:100%`), so all edges render correctly. The parent `canvas` keeps `overflow:hidden` so only the viewport region is visible; panning via CSS transform scrolls the scene within the viewport.
