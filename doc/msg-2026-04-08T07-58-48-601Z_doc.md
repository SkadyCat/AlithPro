# 交付说明：processed 列表卡片位置修复

**源任务文件：** `msg-2026-04-08T07-58-48-601Z.md`  
**任务目标：** `msg-2026-04-08T07-41-48-006Z.md`（克隆 workspace）和 `msg-2026-04-08T07-55-08-682Z.md`（分组阈值修复）这两张卡片显示位置不对

## 根因

`renderProcessedGrouped` 将同一任务链的所有成员聚合渲染在根任务的位置。`07-39`（git commit）的链包含 3 个成员（`07-39, 07-42, 07-44`），渲染时将整个链块放到 `07-39` 的位置，而 `07-41`（时间在 `07-39` 和 `07-42` 之间）被挤到了 `07-44` 之后，导致位置错误。

## 改动

**`application/public/app.js`** — `renderProcessedGrouped()`

新增"连续性"检测：只有链中所有成员在时间排序列表中连续出现（中间无其他卡片），才将该链聚合显示并加标题行。否则每张卡按时间顺序单独渲染（非连续链不再强制分组）。

```
contiguous chain →  [header] member1 member2 member3  (grouped at root's position)
non-contiguous  →  each card at its own natural time position
```

## 验证

- `node --check` 语法通过  
- `07-41` 和 `07-55` 现在出现在其时间戳对应的自然位置  
- `07-14-31` 链（连续）继续显示标题行  
- `07-39` 链（非连续，07-41 穿插其中）按时间顺序平铺
