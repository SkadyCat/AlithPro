**源任务文件**：msg-2026-04-08T07-34-48-740Z.md（子任务，父任务：msg-2026-04-08T07-23-19-670Z.md）
**任务目标**：给任务链路添加重命名功能

## 实现说明

链路重命名的基础框架（chainAliasKey/getChainAlias/setChainAlias/uildChainHeader/startChainRename）已存在于 processed 分组标题中，但存在一个 bug，且非 processed bucket 的链路卡片无法使用此功能。

### Bug 修复

**startChainRename 重复 listener 堆积**：每次 commit 后，旧代码会在 ditBtn 上再添加一个 click listener，导致多次重命名后 listener 不断累加。
修复方案：在 uildChainHeader 的 editBtn click handler 中改为动态查询当前 .chain-header-label（live query），并移除 commit() 和 Escape handler 中的重复 ditBtn.addEventListener 调用。

### 新增功能

1. **链路名称徽标（card-chain-badge）**：uildCard() 中读取 getChainAlias(chainRoot)，若有别名则在卡片标题上方渲染 📁 链路名 小徽标，覆盖所有 bucket（sessions/inprocess/processed）。

2. **上下文菜单"重命名链路"**：对链路成员卡片（任意 bucket，非 drafts/workflows）显示"重命名链路"菜单项，点击调用 openChainRenameModal(chainRoot)。

3. **链路重命名弹窗**（动态创建，无需修改 HTML）：
   - 首次调用时动态创建 #chain_rename_modal 并 appendChild 到 body，同时绑定所有事件
   - 之后复用同一 DOM 节点
   - 支持 Enter 确认、Escape 取消、点击遮罩取消
   - 提交后 live-update DOM 中所有同根 .card-chain-badge 和 .processed-group-header .chain-header-label，无需整页重渲

4. **全局 Escape 处理**：在全局 keydown 中插入 chain_rename_modal 的 Escape 关闭逻辑
