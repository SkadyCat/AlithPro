# 交付说明：Tab 修复 + 思维导图拆分为独立 JS

## 源任务
`msg-2026-04-08T18-10-53-911Z.md` — Tab 键有时不创建子节点 + 拆分思维导图功能到独立文件

## 改动

### 1. 拆分 mindmap.js（核心改动）
- **新文件** `application/public/mindmap.js`：从 `app.js` 提取全部思维导图引擎代码（~330 行），封装在 IIFE 中
  - 状态 `_dm`、渲染、节点 CRUD、拖拽、平移缩放、右键菜单、键盘快捷键
  - 自行绑定 canvas 上的 mousedown/wheel/contextmenu/keydown 事件
  - 通过 `window.MindMap` 暴露公共 API：`{ state, parse, serialize, toMarkdown, render, ensureGradient, hideCtx, canvas }`
- **app.js** 移除 ~330 行内联思维导图代码，替换为 7 行 `window.MindMap` 引用
- **index.html / dev.html** 添加 `<script src="/mindmap.js" defer>` 在 app.js 之前

### 2. Tab 键修复
- **根因**: Tab 键有时被浏览器焦点导航截获（canvas 失焦时 keydown 不触发）
- **修复**: 
  - 节点 mousedown 追加 `canvas.focus()` 确保选中节点后 canvas 重获焦点
  - Tab handler 追加 `e.stopImmediatePropagation()` 彻底阻止浏览器 Tab 行为
