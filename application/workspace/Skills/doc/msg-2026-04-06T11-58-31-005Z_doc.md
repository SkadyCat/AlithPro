## 源任务
`msg-2026-04-06T11-58-31-005Z.md` — play_anim 参数类型错误 bug 修复

## 问题
`anim_comp.lua:56` 调用 `self.entity.actor:play_anim(anim_name)` 时，只传了 `name` 参数，缺少 AngelScript 函数签名所需的 `rate` (float) 参数：
```
void play_anim(FString name, float rate = 3.0f, float& time = 0.f)
```

UnLua 绑定不支持 AS 的默认参数值，导致报错：
```
Error: Invalid parameter type calling ufunction : play_anim, parameter : 1, error msg : number needed but got no value
```

## 修复
`Content/Script/com/components/anim_comp.lua` line 56:
- 旧：`self.entity.actor:play_anim(anim_name)`
- 新：`self.entity.actor:play_anim(anim_name, 3.0)`

补充了 `rate=3.0` 参数（与 AS 默认值一致）。同文件 line 70 的 `do_play_anim` 已经传了 `rate=2`，不受影响。

## 验证
- 代码分析确认 AS 签名 `play_anim(FString, float, float&)` 需要至少两个参数
- 修复与 do_play_anim (line 70) 的调用模式一致
- 需在 PIE 中验证：按技能键 → rush_down 和 magic_release 动画不再报错
