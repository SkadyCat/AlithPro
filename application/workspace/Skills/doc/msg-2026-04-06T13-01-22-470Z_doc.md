# 交付说明 — 环形弹道未创建 bug 修复（CopySkillInfo）

## 源任务文件
`msg-2026-04-06T13-01-22-470Z.md`

## 目标
分析日志并修复环形分裂弹道未创建的问题。

## 日志分析
```
OriginalSkillInfo copied, skillLinks count= 13
DoSkillByCustomArray primary missing: 0  (×4)
Bullet ring release: created 4 projectiles at radius 200.0
```
- skillLinks 有 13 个元素，但所有 FSkillLink.type = 0（默认值）
- AS 端 DoSkillByCustomArray 读取 skillLinks[0].type=0 → GetUMagicAbility 找不到 → "primary missing: 0"
- 环形弹道"创建"了但因为 primary 无效所以实际上跳过了

## 根因
Task 36 的 `TArray:Add()` 修复虽然增加了元素数量，但 UnLua 的 TArray:Add(struct_ref) 对 FPoint2D/FSkillLink 结构体只创建默认值（type=0），**不会拷贝源结构体的字段**。这是 UnLua 处理嵌套 TArray<FStruct> 的已知限制。

## 改动

### 1. Bullet.as — 新增 CopySkillInfo 辅助函数
```angelscript
UFUNCTION(BlueprintCallable)
void CopySkillInfo(const FSkillDataInfo& SourceInfo)
{
    OriginalSkillInfo = SourceInfo;
}
```
在 AS 侧执行 struct 深拷贝，绕过 UnLua 的 TArray 限制。

### 2. skill_comp.lua — 使用 CopySkillInfo
```lua
-- 旧：手动 Add() 循环（struct 字段丢失）
-- 新：调用 AS 辅助函数
bullet:CopySkillInfo(train)
```

## 验证
- `CopySkillInfo` 是 BlueprintCallable UFUNCTION，UnLua 可调用
- AS 的 struct 赋值 `=` 会正确深拷贝嵌套 TArray
- 调用后打印 skillLinks:Num() 确认拷贝成功
- 需在 UE 运行时验证 DoRingRelease 不再报 primary missing

## 风险
- 需要 UE 热编译或重启 AS 才能加载新的 CopySkillInfo 函数
- 如果 train 作为 UFUNCTION 参数传递时 UnLua 的 marshal 也有问题，则需要进一步调整
