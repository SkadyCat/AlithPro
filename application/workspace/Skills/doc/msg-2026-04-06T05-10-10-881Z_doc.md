### 核心标识符

技能系统使用 **AssetID (int64)** 作为全局唯一标识，而非 GameplayTag。AssetID 在 `UAsAsset.OnSave()` 中首次生成（`FDateTime::UtcNow().ToUnixTimestamp()`），并在所有映射表和数据结构中统一使用。

### SkillManager — 全局子系统

`USkillManager : UBaseSubSusystem`

**初始化流程（OnInit）：**
1. `UMagicTool::FindAllAssets()` 收集引擎中所有 UMagicAbility 资产
2. 按 `ReleaseType` 分类写入三张映射表：
   - `AbilityMap<int64, UMagicAbility>` — 全量
   - `PrimaryAbilityMap<int64, UMagicPrimary>` — 主动技能
   - `PassiveAbilityMap<int64, UMagicPassive>` — 被动技能
3. 调用 `Ins.DoManagerSync("SkillManager", this)` 注册到全局实例

**释放技能流程（DoSkill）：**
1. 从角色的 `skill_comp_skill_data` 读取技能链 `FSkillDataInfo`
2. 取 `skillLinks[0].type` 作为主动技能 AssetID
3. `GetUMagicAbility()` 查表 → 强转 `UMagicPrimary`
4. `SpawnBullet()` 创建 ABullet 实体（设 Niagara 特效 + 朝向）
5. 遍历 `CastAbility.AbilityClasses` 调用每个被动的 `Process()` 修改子弹
6. 遍历 `skillLinks[1..]` 查找额外被动并逐一 `Process()`

**DoSkillByCustomArray：** 支持自定义技能链数组，流程与 DoSkill 相同但接收 `TArray<FSkillDataInfo>` 参数。

### ABullet — 子弹实体

`ABullet : AEntity`

**关键组件：**
- `UProjectileMovementComponent` — 抛射物运动
- `UNiagaraComponent` — 视觉特效
- `USphereComponent` — 碰撞检测
- `USplineComponent` — 曲线路径（用于追踪弹道）
- `TArray<UBulletBehavior> Behaviors` — 弹道行为列表（工厂模式）

**Tick 循环：** 遍历 `Behaviors` 数组依次调用 `TickBehavior()`。

**碰撞处理（OnCompOverlap）：**
1. 先让所有 Behavior 的 `OnOverlap()` 处理，任一返回 true 则消费事件
2. 否则走默认逻辑：判断友军/敌军（EEntityType），生成 FHitInfo 并通知 `ListenSkillHit`，最后 `DestroyActor`

### 弹道行为工厂

`bulletbehavior/` 目录下每个行为独立实现，**互不继承、互不调用**。

基类：`UBulletBehavior : UObject`
- `Init(ABullet)` — 初始化
- `TickBehavior(ABullet, float)` — 每帧更新
- `OnOverlap(ABullet, AActor, FHitResult) → bool` — 碰撞拦截

| 行为 | 关键逻辑 |
|---|---|
| Bounce | 在 Tick 中检测地面高度，到达地面时反弹、递减弹跳高度 |
| BounceExplode | 弹跳 + 碰撞时根据 TriggerTypes (EEntityType) 决定是否爆炸 |
| Curve | 沿 Spline 曲线飞行（追踪弹） |
| Tracking | 持续跟踪 TargetActor 位置，调整速度方向 |
| Split | 碰撞时分裂为 N 个子弹（调用 SkillManager.DoSkillByCustomArray） |
| Ring | 初始化时在环形方向上生成多个子弹 |
| Accelerate | 每帧叠加速度倍率 |

### skill_comp.lua — Lua 技能组件

运行在 UnLua 层，挂载在角色 Entity 上。

**数据存储：**
- `unlocked_skills` 属性 — `{uuid: {asset_id, data, unlock_time}}`，支持 `sync_to_client` + `save_to_db`
- `skill_data` — 引用角色 Actor 上的 `FSkillDataList`（技能链网格）
- `equip_info` — 装备槽位

**核心 API：**
| 方法 | 说明 |
|---|---|
| `unlock_skill(asset_id)` | 解锁技能，防重复，返回 UUID |
| `get_unlocked_skill(uuid)` | 查单个 |
| `get_all_unlocked_skills()` | 查全部 |
| `update_unlocked_skill(uuid, data)` | 更新自定义属性 |
| `remove_unlocked_skill(uuid)` | 移除 |
| `init_skill()` | 首次登录初始化技能链网格（4×12 随机路径，17 格） |
| `set_skill_info(data, tag)` | 设置技能链某格的 AssetID |
| `do_skill(params)` | 发起释放技能请求（构建 AnimMsg → G.SkillManager:Create） |
| `calc_arr(x, y, n)` | Fisher-Yates 随机路径算法，生成 n 格连通路径 |

**技能链网格模型：**
角色拥有 `FSkillDataList`，包含多条 `FSkillDataInfo`，每条由 `TArray<FPoint2D> skillLinks` 组成。每个 `FPoint2D` 有 `(x, y, type)`，其中 `type` 存储 AssetID。第一个 link 必须是 Primary 技能，后续为 Passive 修饰。
