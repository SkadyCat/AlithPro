# 交付说明：无效石板节点清除功能

## 源任务文件
`msg-2026-04-06T14-26-04-225Z.md`

## 任务目标
在 `panel_skill_connect.lua` 中，任意节点（包括主技能和被动技能槽位）都可以选择"无效石板"。选择后通知服务器清除该节点的链接（type 设为 0）。

## 修改内容

### panel_skill_connect.lua
1. **build_filter** (第 124-138 行): 在 match 函数中新增 `abilityType == "invalid"` 的判断，使无效石板在任意槽位（primary/passive）都始终处于可选状态，不受 releaseType 和已使用 ID 的过滤限制。
2. **select_skill** (第 158-168 行): 检测所选技能的 `abilityType`，如果是 `"invalid"`（即无效石板），则将传给服务器的 assetId 替换为 `0`，触发清除逻辑。

### 服务端 set_skill_info 已有兼容逻辑
- `skill_comp.lua:set_skill_info` 中：当 `tag_name = 0` 时，`skill_registry.get(0)` 返回 nil → 跳过去重检查 → 直接 `val.type = 0`（清除链接）
- 无需修改服务端代码

## 验证方法
1. 打开技能连接面板（panel_skill_connect）
2. 点击任意节点 → 打开技能书（panel_skill_screen）
3. 无效石板应显示为可选（enabled）状态
4. 选择无效石板 → 该节点 type 应变为 0，UI 刷新后显示为空槽位

## 剩余风险
- 清除主技能节点（index=1）后技能链将无法释放 — 这是预期行为（用户可重新选择）
