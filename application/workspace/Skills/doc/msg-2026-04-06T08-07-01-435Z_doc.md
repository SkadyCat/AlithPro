# 交付说明 — 重构技能系统使用 Lua 数据

## 源任务
`msg-2026-04-06T08-07-01-435Z.md`

## 目标
移除之前使用 DA（DataAsset）作为技能节点数据源的方式，改用 Lua table 数据作为主要数据源。

## 改动

### server.js
- `/api/skills` 不再返回 `instances` 字段（DA 实例数据）
- `/api/refresh` 同时刷新 Lua 树和类定义缓存
- DA 扫描代码保留内部使用（export 端点仍可用），但不再作为主 API 输出

### app.js (前端)
- 默认视图从 `instances` 改为 `luaTree`（Lua 数据树）
- 移除 `renderInstanceMenu()`、`selectInstance()`、`renderInstanceDetail()` 三个函数
- 移除 `switchToInstanceAndSelect()` 跨视图导航
- 新增 `switchToLuaAndSelect()` 用于跨视图导航到 Lua 节点
- 视图切换简化为两态：`luaTree`（数据树）↔ `classes`（类定义）
- 移除类详情页中的「关联实例」区域
- `instances` 变量已移除

### index.html
- 切换按钮初始文本不变（"类定义"），与默认 luaTree 视图一致

## 验证
- `node --check` 语法检查通过
- `/api/skills` 返回 158 节点，无 instances 字段
- `/api/lua-tree` 返回 35 文件、15 技能实例
- 页面正常加载（HTTP 200）

## 剩余风险
- DA export 端点 `/api/export/:name` 仍依赖内部 DA 缓存，如后续不需要可进一步清理
