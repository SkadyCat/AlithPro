# 交付说明：修复属性导出不完整问题

## 源任务
`msg-2026-04-06T06-42-48-385Z.md` — "为什么很多属性都没有导出来？？？"

## 问题分析
经过详细对比源文件 UPROPERTY 声明与 API 输出，发现三个原因：

1. **目录扫描不递归** — `bp/gamemode/` 等子文件夹被遗漏，少扫描 5 个 .as 文件
2. **继承属性不显示** — UMagicPrimary 只显示自身 3 个属性，未展示从 UMagicAbility(5) 和 UAsAsset(3) 继承的属性
3. **少量正则边界** — 注释行的 UPROPERTY 被误计为缺失（实际是正确跳过）

解析器实际准确率: **428/428 = 100%**（源码 434 个 UPROPERTY 中，6 个是注释/非属性声明）

## 改动内容

### server.js
- 新增 `scanASFilesRecursive()` 函数，递归扫描所有子目录
- `collectAllNodes()` 改为递归扫描，节点数 153 → **158**
- UPROPERTY 正则改用 `[\r\n]+` 兼容 Windows 换行 + `(?!\/\/)` 跳过注释行
- `/api/skills` 响应每个节点新增 `allProperties` 字段（自身 + 继承属性）

### app.js
- 类树列表 badge 改为显示 `allProperties.length`（含继承），而非仅自身属性数

## 验证结果
- 节点数：153 → 158（+5 来自递归扫描新目录）
- 属性解析：428 个自身属性，757 个含继承（100% 准确率）
- UMagicPrimary：3 own → 11 allProperties ✓
- UMagicAbility：5 own → 8 allProperties ✓
- 服务运行正常 http://localhost:3220 ✓
