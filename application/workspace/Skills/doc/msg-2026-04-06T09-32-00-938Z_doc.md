# 交付说明 — 子弹初始位置修正为鼠标位置

## 源任务
`msg-2026-04-06T09-32-00-938Z.md`

## 目标
修复子弹创建位置不在鼠标光标处的问题。

## 根因
`do_skill()` 使用 `release_pos`（角色释放位置）作为 SpawnActor 坐标。但 `entity.lua` 的动画回调只设置了 `CursorPos`/`Dir`/`master`，`ReleasePos` 依赖 AnimMsg 结构体由动画系统填充，可能为默认值或角色脚下。

## 修复

**文件**: `Content/Script/com/components/skill_comp.lua` — `do_skill()`

1. **生成位置** → 使用 `cursor_pos`（鼠标位置），Z 轴使用射线追踪到的地面高度
2. **兜底逻辑** → 如果 `release_pos`/`cursor_pos` 缺失，回退到角色位置
3. **方向计算** → 始终从角色位置指向鼠标位置
4. **射线追踪** → 不再有条件判断，cursor_pos 有兜底保证非 nil

## 验证
- 生成坐标 = `FVector(cursor_pos.X, cursor_pos.Y, ground_height)`
- 方向 = `Normalize(cursor_pos - release_pos)`，Z=0
- 需运行时验证子弹是否出现在鼠标点击位置

## 风险
- 对于"从角色发射"类技能，此行为可能不符预期（当前所有技能统一在鼠标位置生成）
