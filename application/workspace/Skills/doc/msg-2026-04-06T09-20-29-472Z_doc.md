# 交付说明 — 弹道持续移动 Bug 修复

## 源任务
`msg-2026-04-06T09-20-29-472Z.md`

## 目标
修复弹道（子弹）创建后不断移动、永不销毁的问题。

## 根因分析

对比 AS 原版 `SkillManager.DoSkill()` 和 `Bullet.as`，发现三处问题：

| # | 问题 | AS 原版行为 | Lua 实现（修复前） |
|---|------|-------------|---------------------|
| 1 | 初始速度 | `Velocity = FVector::ZeroVector`，由被动 Process() 设置 | `Velocity = dir * speed`，直接设满速 |
| 2 | 生命周期 | `System::SetTimer(OnLifeEnd, LifeSpan)` 定时器销毁 | `bullet.LifeSpan = 5`（属性赋值，不触发销毁） |
| 3 | 生成旋转 | `SpawnActor(class, pos, FRotator::ZeroRotator)` | `SpawnActor(class, pos, dir:ToRotator())`（ToRotator 可能不存在于 UnLua） |

## 修复内容

**文件**: `Content/Script/com/components/skill_comp.lua` — `do_skill()` 方法

1. **生成旋转** → `UE.FRotator(0, 0, 0)`（ZeroRotator）
2. **初始速度** → `UE.FVector(0, 0, 0)`（零速度，匹配 AS）
3. **被动链执行后回退** → 如被动未设速度，才用 `dir * speed` 作直线弹道回退
4. **朝向设置** → `UE.UKismetMathLibrary.Conv_VectorToRotator(dir)`（替代不存在的 `ToRotator()`）
5. **生命周期** → `bullet:SetLifeSpan(life)`（方法调用，触发 UE 定时销毁）
6. **移除** `K2_SetActorLocation` 重复调用（SpawnActor 已设位置）

## 验证
- 代码逻辑与 AS 原版 DoSkill() + SpawnBullet() 对齐
- SetLifeSpan 是 UE AActor 标准方法，UnLua 可正确调用
- Conv_VectorToRotator 是 KismetMathLibrary 标准函数
- 需运行时验证子弹是否按 LifeSpan 定时销毁

## 风险
1. 被动 Process() 尚未在 Lua node 类实现，当前全部走回退直线弹道
2. `bullet.OriginalSkillInfo = train` 赋值是否可行取决于 BP_Bullet C++ 属性类型
