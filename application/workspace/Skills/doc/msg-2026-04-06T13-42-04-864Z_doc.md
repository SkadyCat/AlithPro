# 交付说明 — 技能链阻塞机制

## 源任务
`msg-2026-04-06T13-42-04-864Z.md` — 技能节点链是阻塞的，分裂完之后才能 process 下一个节点。需要加入 callback next node 机制并兼容当前技能节点。

## 目标
为技能被动链引入阻塞 (blocking) 机制：遇到阻塞节点时停止父弹处理，剩余节点由子弹行为完成后通过 AS 回调路径传递给子弹。

## 架构设计

### 原有流程（全同步）
```
父弹: StraightShot → RingSplit → [后续被动] → 全部同步处理
环形触发: DoRingRelease → 子弹继承全链（按类型过滤 Ring）
```

### 新流程（阻塞链）
```
父弹: StraightShot → RingSplit(blocking=true) → STOP
          ↓ 记录 ChainBlockedAt = 阻塞节点 0-based 索引
环形触发: DoRingRelease → 子弹继承全链（按索引过滤阻塞节点）
子弹:  StraightShot → [后续被动] → 继续处理
```

### 关键改进
1. **索引过滤**替代**类型过滤**：DoRingRelease 按 `Bullet.ChainBlockedAt` 索引跳过特定阻塞节点，而非按 `Cast<UMagicProjectileRing>` 跳过所有同类型节点。支持链中出现多个同类被动的场景。
2. **process() 新增 chain_index 参数**：所有被动节点的 `process(actor, bullet, params, chain_index)` 现在接收链索引，阻塞节点可据此记录自身位置。

## 验证

- Lua 侧：阻塞循环逻辑正确，非阻塞节点正常 process，阻塞节点 process 后 break
- AS 侧：DoRingRelease 按索引跳过阻塞节点，子弹继承剩余链
- **需 UE 热编译**加载 Bullet.as 新属性 `ChainBlockedAt` 和 RingBulletBehavior.as 改动

## 剩余风险

1. 需 UE AS 热编译才能生效
2. 其他行为类（SplitBulletBehavior, BounceBehavior 等）暂未改为索引过滤，后续如需阻塞需同步更新
3. 子弹通过 AS `DoSkillByCustomArray` 路径继续执行，Lua 侧目前不参与子弹的被动处理
