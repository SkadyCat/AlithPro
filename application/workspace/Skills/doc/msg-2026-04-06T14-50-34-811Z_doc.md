# 交付说明：分裂类被动互斥 — 子弹道不再继承其他分裂逻辑

## 源任务文件
`msg-2026-04-06T14-50-34-811Z.md`

## 任务目标
分裂类被动（Split、RingSplit）触发后，子弹道不应再继承任何其他分裂逻辑。即 Split 和 RingSplit 互斥：一旦其中一个触发，子弹道的技能链中不再包含另一个。

## 修改内容

### 1. SplitBulletBehavior.as — DoSplit（第 142-153 行）
- 移除旧的 index-based 过滤（`ChainBlockedAt`）
- 改为 type-based 过滤：遍历所有被动，`Cast<UMagicProjectileSplit>` 和 `Cast<UMagicProjectileRing>` 匹配的全部跳过
- 子弹道只继承非分裂类被动（如 StraightShot、Accelerate 等）

### 2. RingBulletBehavior.as — DoRingRelease（第 148-159 行）
- 同上：移除 index-based 过滤，改为 type-based 过滤
- 子弹道只继承非分裂类被动

### 预期效果
| 技能链 | 父弹 | 子弹道 |
|---|---|---|
| Primary + StraightShot + Split + RingSplit | Split 触发（blocking） | 只有 Primary + StraightShot（无 RingSplit） |
| Primary + StraightShot + RingSplit + Split | RingSplit 触发（blocking） | 只有 Primary + StraightShot（无 Split） |

## 验证方法
1. UE AS 热编译
2. 装备 Split + RingSplit 技能链
3. 释放技能 → 只看到一次分裂效果，子弹道不会再次分裂

## 剩余风险
- 需要 UE AS 热编译
- 后续新增分裂类被动（如 Bounce 等）需在两处过滤中加入对应 Cast 检查
