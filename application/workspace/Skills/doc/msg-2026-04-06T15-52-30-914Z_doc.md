# 交付说明 — 辅助石节点属性生成

## 源任务
`msg-2026-04-06T15-52-30-914Z.md` — 给生成的500个辅助石添加具体节点属性

## 目标
生成的辅助石只有分类名称（trajectory / split / element 等），缺少具体的游戏参数。需要根据各分类特性，为每个辅助石添加差异化的节点属性。

## 改动

### generate_skill_catalog.js
1. 为 10 个 PASSIVE_CATEGORIES 各添加 `nodeProps` 模板：
   - trajectory: trajectoryType, angleOffset, curveIntensity, trackingRadius, updateInterval
   - split: splitCount, splitAngle, triggerDistance, bTriggerOnCollision, damageInheritance
   - element: elementType, damageBonus, dotDuration, dotTickInterval, effectRadius
   - speed: speedMultiplier, accelerationRate, maxSpeedCap, decelerationDelay, bAffectsProjectile
   - range: rangeMultiplier, falloffStart, falloffEnd, bPiercing, maxBounces
   - duration: durationMultiplier, tickRate, maxStacks, bRefreshable, decayRate
   - damage: damageMultiplier, critChance, critMultiplier, armorPenetration, bSplashDamage
   - effect: effectType, effectChance, effectDuration, effectIntensity, bStackable
   - defense: defenseType, damageReduction, blockChance, reflectPercent, shieldAmount
   - control: controlType, controlDuration, controlChance, slowPercent, knockbackForce
2. 新增 `resolveNodeProps(template, index)` 函数，根据 index 模运算在 min-max 范围内递增取值，枚举循环，布尔按模式切换
3. `generatePassives()` 调用 resolveNodeProps 将解析后的属性注入每个技能对象
4. `generateCatalogJSON()` 被更新为导出 nodeProps 到 JSON

### projectile-design.html
5. 添加 `nodePropsToTags()` 辅助函数将 nodeProps 对象转为 `key:value` 标签
6. 辅助石渲染 mapping 中加入 `props: nodePropsToTags(s.nodeProps)` 以在页面显示

## 风险
- 属性值为程序化生成的示例值，游戏平衡调整时需修改模板参数
- 属性目前仅作为 Lua 类属性存在，运行时读取依赖 skill_comp.lua 解析逻辑
