# 交付说明：Split → RingSplit 执行顺序修复

## 源任务文件
`msg-2026-04-06T14-42-07-367Z.md`

## 任务目标
修复 Split（分裂）和 RingSplit（环形分裂）的执行顺序：Split 应先触发（弹道分裂为多个子体），每个子体再触发 RingSplit（环形释放）。

## 问题分析
之前 `MagicProjectileSplit` 没有 `process()` 方法且 `blocking = false`，导致：
1. Lua 被动循环跳过 Split（无 process → 不添加 SplitBulletBehavior）
2. 直接到 RingSplit（有 process → 添加 RingBulletBehavior，blocking → break）
3. 父弹只有 Ring 行为，Split 从未生效

## 修改内容

### 1. MagicProjectileSplit.lua（Content/Script + Script/ 双目录同步）
- 新增 `blocking = true` — 分裂是阻塞节点
- 新增 `process()` 方法 — 加载 `BulletSplitBehavior` AS 类，调用 `bullet:AddBehavior()`，设置 SplitCount/SplitAngle/TriggerDistance/bTriggerOnCollision

### 2. SplitBulletBehavior.as — DoSplit 改为索引过滤
- 移除旧的 type-based 过滤（`Cast<UMagicProjectileSplit>`）
- 改为与 RingBulletBehavior 一致的索引过滤：读取 `Bullet.ChainBlockedAt`，跳过该索引
- 子弹道继承完整技能链（除去当前分裂节点），RingSplit 等后续节点保留

### 预期效果（技能链: Primary → StraightShot → Split → RingSplit）
1. 父弹: StraightShot(速度) → Split(blocking, 添加分裂行为) → break
2. 父弹飞行 200 距离后触发分裂 → 3 个子弹道
3. 每个子弹道链: Primary + StraightShot + RingSplit（Split 已被索引过滤）
4. 子弹道触发 RingSplit → 每个释放 4 个环形投射物
5. 最终: 3×4 = 12 个终极投射物（有限，不会级联）

## 验证方法
1. UE AS 热编译
2. 装备链: 主技能 + 直射 + 分裂 + 环形分裂
3. 释放技能 → 弹道先分裂为 3，每个再环形释放 4
4. 检查日志无 "passive missing: 0" 级联

## 剩余风险
- 需要 UE AS 热编译
- Bounce/BounceBlast 等被动也可能需要 blocking + process()，后续按需添加
