# 交付说明：msg-2026-04-06T07-23-56-898Z

## 源任务
`msg-2026-04-06T07-23-56-898Z.md` — "我重启服务了，再试一下，还不行的话 用c++来做"

## 目标
UE 重启后重试 McpFlask `/ue/inspect-da` 端点获取 DA 资产的实际属性值。

## 诊断与解决

### 问题链
1. **Python `get_editor_property()` 无法读取 AngelScript 定义的属性** — Python binding 不暴露 AS UPROPERTY
2. **McpFlask tick 回调在热重载后失效** — `run_server.py` 注册的 `_tick_drain` 在 `app.py` reload 后不再被调用
3. **C++ 函数从 Flask 线程调用会死锁** — `UMagicTool::ExportAllAssetsToJson` 内部加载资产需要 GameThread

### 解决方案
1. **app.py 自注册 Slate tick 回调** — 在模块顶层注册 `register_slate_post_tick_callback`，每次热重载后自动刷新，彻底修复 tick 失效问题
2. **调用 C++ `UMagicTool::ExportAllAssetsToJson`** — 通过 tick 队列在 GameThread 执行，使用 `FJsonObjectConverter::UStructToJsonObject` 正确序列化所有 UPROPERTY（含 AS 定义的属性）
3. **UTF-16 编码处理** — UE 写入的 JSON 是 UTF-16 LE (BOM)，在 McpFlask 和 Skills server 中均做了编码检测

## 完成内容

### McpFlask (app.py)
- 新增 app.py 顶层自注册 Slate tick callback（热重载安全）
- `_inspect_da_sync` 改为通过 tick 队列调用 C++ ExportAllAssetsToJson
- 修复 JSON 读取编码为 UTF-16
- 新增 `drain_ue_cmd_queue()` + `/ue/diag` 诊断端点

### Skills 可视化器

- `server.js`：`buildInstanceData` 正确解析 `{Assets: [...]}` 格式
- `app.js`：统一 UE 属性值显示（合并 exportedData.Properties 和 ueProperties）
- 长值自动折叠显示

## 关键发现
- McpFlask 热重载 tick 失效的根因：`run_server.py` 的 `_tick_drain` 在 `importlib.reload(app)` 后停止被调用（可能是 Slate callback handle 失效）
- 解决方案：在 app.py 模块级别自行注册 tick callback，每次 reload 自动注册新的
- `FJsonObjectConverter::UStructToJsonObject` 是读取 AngelScript UPROPERTY 的正确 C++ 路径
