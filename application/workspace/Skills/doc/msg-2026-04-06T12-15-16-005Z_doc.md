# 交付说明 — 直射被动不移动子弹修复

## 源任务文件
`msg-2026-04-06T12-15-16-005Z.md`

## 目标
修复添加直射（StraightShot）被动后，子弹不向前移动的问题。

## 根因分析
- `skill_comp.lua:268` 在被动处理循环中检查 `if passive and passive.process then passive:process(...)`
- Lua 被动类（StraightShot、Bounce 等）全部只有数据属性，**没有 `process()` 方法**
- `process()` 逻辑原本在 AngelScript `UMagicStraitBall.Process()` 中：设置子弹速度和方向
- 因此 `passive.process` 始终为 nil，循环体不执行，子弹速度保持 (0,0,0)

## 改动
### MagicStraitBall.lua（两个目录同步更新）
- `Content/Script/com/ability/node/.../MagicStraitBall/MagicStraitBall.lua`
- `Script/com/ability/node/.../MagicStraitBall/MagicStraitBall.lua`

添加 `process(actor, bullet, params)` 方法，移植自 AS 原版逻辑：
1. 从 `params.Dir` 获取方向（兼容 AnimMsg struct 大小写）
2. 如果 Dir 为 nil，回退到 `CursorPos - ReleasePos` 或角色前方
3. 检查方向向量长度：> 1.1 则含速度信息，否则使用 `self.speed`（StraightShot=100）
4. Z 轴归零，`Vector_Normalize` 归一化
5. 设置 `ProjectileMovement.Velocity = norm_dir * final_speed`
6. 设置 `MaxSpeed=3000`, `InitialSpeed=3000`, `GravityScale=0`
7. 设置子弹旋转 `Conv_VectorToRotator(vel)` → `K2_SetActorRotation`
8. 打印调试日志 `[SKILL_DBG]`

### 继承关系
StraightShot → MagicStraitBall → MagicPassive → MagicAbility
- StraightShot 通过 Lua metatable `__index` 继承 `MagicStraitBall:process()`
- 调用时 `self.speed` 解析为 StraightShot.speed = 100

## 验证
- 代码逻辑与 AS 原版 `UMagicStraitBall.Process()` 一致
- 使用 codebase 一致的 `Vector_Normalize` 而非 `Normal`（避免 API 不可用）
- 两目录文件已同步，仅 require 路径不同

## 风险/注意
- CurveHoming（曲线追踪）继承自 MagicStraitBall，也会获得 `process()` — 其 AS 原版有额外逻辑（FindNearestActor + SetBehavior），后续需覆盖
- 其他被动类（Bounce、Split 等）的 `process()` 尚未实现
- 需在 UE 运行时验证子弹确实向前移动
