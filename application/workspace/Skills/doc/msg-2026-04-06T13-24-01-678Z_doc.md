# 交付说明 — 环形分裂 CopySkillInfo bug 修复

## 源任务
`msg-2026-04-06T13-24-01-678Z.md` — 环形分裂的 bug 还是没有修好，观察日志，然后分析问题。

## 目标
分析 UE 日志，定位环形分裂 (RingSplit) 技能失败的根本原因并修复。

## 问题分析

日志关键错误：
```
Error: Invalid parameter type calling ufunction : CopySkillInfo, parameter : 0,
error msg : userdata needed but got table
```

**根本原因**：`skill_comp.lua` 中 `train = self.skill_data.skillDataInfos:get(1)` 返回的是一个 `property_read` **代理 Lua 表**（通过 `com.proxy.property_arr:get()` 创建），而非原始 UE userdata。`CopySkillInfo` UFUNCTION 期望 `const FSkillDataInfo&`（UE struct userdata），收到 Lua table 后调用失败，导致 `bullet.OriginalSkillInfo.skillLinks` 始终为空。

代理链路追踪：
1. `self.skill_data` → `component.__getattr` → `property_read` 代理（`_val` = 原始 UE struct）
2. `.skillDataInfos` → `property_read.__getattr` → `property_arr` 代理（`_val` = 原始 TArray）
3. `:get(1)` → `property_arr.get()` → `TArray:GetRef(1)` → 再包装为 `property_read` 代理
4. 最终 `train` 是 Lua table，真正的 UE struct 在 `train._val` 中

## 改动

| 文件 | 行 | 改动 |
|---|---|---|
| `Content/Script/com/components/skill_comp.lua` | 255-256 | `bullet:CopySkillInfo(train)` → `bullet:CopySkillInfo(train._val)` |

## 验证

- 修复后 `CopySkillInfo` 将接收到 `FSkillDataInfo` userdata（而非 proxy table）
- `bullet.OriginalSkillInfo.skillLinks` 将正确包含技能链数据
- `BulletRingBehavior.DoRingRelease()` 将能读取 skillLinks 并创建子弹

**需 UE 热编译或重启后验证**：CopySkillInfo 是新增的 AS UFUNCTION（上一个任务添加），需要 UE 加载后才能调用成功。

## 附加发现

1. `StraightShot.speed = 100`（DA 数据）— 速度较低（≈1 m/s），但与数据资产一致
2. `RingSplit.triggerDistance = 1`（DA 数据）— 环形几乎瞬间触发，但这是 DA_Ring 的原始设定

## 剩余风险

- 如果 UE 未热编译 Bullet.as 中的 `CopySkillInfo` UFUNCTION，调用仍会失败（函数不存在）
- 修复后需要实际测试确认子弹创建和技能链传递正常
