# 交付说明 — OriginalSkillInfo.skillLinks 为空导致环形弹道未创建

## 源任务文件
`msg-2026-04-06T12-47-49-886Z.md`

## 目标
修复环形分裂触发后 `Bullet.OriginalSkillInfo.skillLinks is empty` 导致弹道未创建的问题。

## 根因分析
日志显示：
- ✅ StraightShot.process() 正常 — speed=100, vel=(72.6, 52.9, 0)
- ✅ RingSplit.process() 正常 — ring behavior added, count=4, radius=200, triggerDist=1
- ❌ `DoRingRelease: Bullet.OriginalSkillInfo.skillLinks is empty` × 4

**原因**：`skill_comp.lua:255` 使用 `bullet.OriginalSkillInfo = train` 直接赋值 UE struct。在 UnLua 中，UE struct 的直接赋值不会深拷贝嵌套的 `TArray<FSkillLink>` (`skillLinks`)，导致子弹的 `OriginalSkillInfo.skillLinks` 为空数组。AS 端 `DoRingRelease()` 读取时发现无技能链数据。

## 改动
### `Content/Script/com/components/skill_comp.lua`
替换简单赋值：
```lua
-- 旧：bullet.OriginalSkillInfo = train
-- 新：手动深拷贝
local orig = bullet.OriginalSkillInfo
orig.skillLevel = train.skillLevel or 1
for i = 1, train.skillLinks:num() do
    orig.skillLinks:Add(train.skillLinks:get(i))
end
```

## 验证
- 打印 `skillLinks count=` 确认拷贝后数量正确
- 需在 UE 运行时验证 DoRingRelease 不再报 skillLinks empty

## 风险
- `TArray:Add()` 对 struct 元素（FSkillLink）的拷贝行为需运行时验证
- 如果 `Add()` 不可用，可能需要尝试 `add()` 或其他 UnLua TArray API
