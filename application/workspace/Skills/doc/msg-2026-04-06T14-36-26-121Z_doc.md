# 交付说明：修复对象池导致的节点 icon 残留

## 源任务文件
`msg-2026-04-06T14-36-26-121Z.md`

## 任务目标
修复 `panel_skill_connect.lua` 中因 UE ListView 对象池复用 widget 导致的节点 icon 残留问题。

## 修改内容

### panel_skill_connect.lua — on_initialize_skill_slot (第 170-207 行)
在函数入口处（进入 `if item.custom_data.data` 判断之前）统一重置 widget 状态：
1. `helpers.set_button_texture(widget.button, nil)` — 清空按钮纹理，避免旧 icon 残留
2. 四个方向箭头 (left/right/top/bottom) 全部设为 Collapsed — 避免旧方向箭头残留
3. `widget.button.OnClicked:Clear()` — 清除累积的点击事件绑定

之后正常流程：有数据时加载正确 icon 和方向箭头，无数据时整个 widget 隐藏。

## 验证方法
1. 打开技能连接面板
2. 滚动 / 切换页签使 slot widget 被池复用
3. 确认空槽位不会显示前一个技能的 icon
4. 确认方向箭头仅出现在有连接方向的节点上

## 剩余风险
- 若 `OnClicked:Clear()` 在当前 UnLua 版本中不存在，可能需要改用其他 API 清除委托
