# 交付说明 — 执行层全面 Lua 化，数据使用 node 类继承体系

## 源任务
`msg-2026-04-06T08-52-28-395Z.md`

## 目标
1. 执行层全部用 Lua 处理，抛弃 AS 逻辑
2. 抛弃 DA (DataAsset) 逻辑
3. 数据使用 `com/ability/node/MagicAsset` 下的 Lua 类继承体系
4. 删除之前手造的 `skill_node_data.lua`

## 改动

### 删除文件
- **`Content/Script/com/ability/skill_node_data.lua`** — 手造数据模块，被 node 类注册表替代

### 新增文件

**`Content/Script/com/ability/node/`** — 36 个 Lua 类文件（从 `Script/com/ability/node/` 复制）
- require 路径已更新为 `com.ability.node.xxx` 格式，可从 UnLua 正确加载
- 保留完整继承链：class.lua → MagicAsset → AsAsset → MagicAbility → MagicPrimary/MagicPassive → 具体技能

**`Content/Script/com/ability/skill_registry.lua`** — 技能注册表
- 加载全部 15 个技能实例（3 主技能 + 12 被动技能）
- 按 `assetId` 索引，通过 `__index` 链自动继承父类属性
- API: `get(id)`, `get_primaries()`, `get_passives()`, `get_random_primary()`, `has_release_type(id, type)`, `get_all()`

### 修改文件

**`Content/Script/com/components/skill_comp.lua`**

| 位置 | 旧代码 | 新代码 |
|------|--------|--------|
| L8 | `require('com.ability.skill_node_data')` | `require('com.ability.skill_registry')` |
| `set_skill_info()` | `skill_node_data.get_ability()` + `ability_type` | `skill_registry.get()` + `abilityType` |
| `init_skill()` | `skill_node_data.get_random_primary()` | `skill_registry.get_random_primary()` |
| `do_skill()` | `G.SkillManager:Create(actor, enum, msg)` | 全 Lua：射线追踪 + SpawnActor + 设置 Trail/Speed/LifeSpan + 被动链 |
| `extract_ability_data()` | 读 snake_case 属性 | 读 node camelCase 属性（`abilityType`, `skillName`, `releaseType`） |
| `unlock_skill()` | `skill_node_data.get_ability()` | `skill_registry.get()` |

**`Content/Script/com/core/entity.lua`**
- L270: `G.SkillManager:DoSkill(self.actor, 0, params)` → `self.skill_comp:do_skill(params)`

**`Content/Script/com/components/ai_comp.lua`**
- L21: `G.SkillManager:DoSkill(self.entity.actor, 0, params)` → `self.entity.skill_comp:do_skill(params)`

### 架构变化

```
之前:  Lua → G.SkillManager:DoSkill() → AS SpawnBullet() → UE Actor
现在:  Lua skill_comp:do_skill() → UE.SpawnActor + UE.UObject.Load → UE Actor
       数据: com.ability.node 类继承体系（不经过 AS/DA）
```

## 风险
1. **UnLua SpawnActor API**: `world:SpawnActor(class, pos, rot)` 的确切签名需运行时验证
2. **BP_Bullet 路径**: 使用 `/Game/BP/BP_Bullet.BP_Bullet_C`，如蓝图路径不同需调整
3. **被动 Process**: 各被动的运行时效果（弹射、加速、追踪等）原由 AS 类实现，Lua 端目前为空壳，需逐个移植
