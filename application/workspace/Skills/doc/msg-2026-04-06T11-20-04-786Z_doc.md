# 迭代 panel_skill_screen 使用 Lua 数据

**源任务**: `msg-2026-04-06T11-20-04-786Z.md`
**目标**: 彻底迭代技能面板相关文件，确保所有属性访问使用 Lua camelCase 数据

## 问题诊断

1. **calc_desc 模板替换失败**: desc 模板用 PascalCase `{Speed}`/`{LifeSpan}`，但 Lua 属性是 camelCase `speed`/`lifeSpan`，原有 fallback 只尝试首字母大写（方向反了）
2. **skill_item_widget.lua 属性名错误**: 仍使用旧 DA snake_case 属性（`.name`, `.description`, `.asset_id`, `.ability_type`, `.life_span`）而非 Lua camelCase
3. **unlocked_skills.lua 图标加载**: 仍通过 `G.SkillManager:GetUMagicAbility()` 获取 DA Icon

## 改动文件

### panel_details.lua — calc_desc 修复
- fallback 改为首字母小写: `{Speed}` → 先试 `data.Speed`(nil) → 再试 `data.speed`(✓)
- 同理 `{LifeSpan}` → `data.lifeSpan`, `{BounceTimes}` → `data.bounceTimes`

### skill_item_widget.lua — 全面重写
- `.name` → `.skillName`
- `.description` → `.desc`
- `.asset_id` → `.assetId`（已无需，直接用 `skill_data.icon`）
- `.ability_type` → `.releaseType`（改为中文数组显示）
- `.life_span` → `.lifeSpan`
- 图标加载简化：直接 `UE.UObject.Load(skill_data.icon)` 而非二次查 registry
- 移除 `.level`、`.damage`、`.radius` 等不存在的属性引用
- 移除冗余 print 日志

### unlocked_skills.lua — 图标加载
- `G.SkillManager:GetUMagicAbility()` → `skill_registry.get()` + `UE.UObject.Load(ability.icon)`

## 验证

- ✅ 全部 UI panel 文件无 G.SkillManager 引用（仅 panel_main_screen + 注释 + 非技能 API 除外）
- ✅ 属性名统一使用 Lua camelCase
- ⚠️ 需 UE 重启验证 UI 显示和模板替换
