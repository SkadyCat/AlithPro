## 调研结论

- 官方安装入口：
  - `https://godotengine.org/download/windows/`
  - 官方文档确认 Godot 支持命令行运行和 `--path` / `--headless` / `--quit-after` 等参数。
- 本次关注的 Godot MCP 方向：
  - `Coding-Solo/godot-mcp`
    - 特点：Node.js / TypeScript 实现，支持启动编辑器、运行项目、获取调试输出、列项目、创建场景与节点操作。
    - 适合当前工作区：因为本仓库已经有 Node.js 环境，接入成本最低。
  - `ee0pdt/Godot-MCP`
    - 特点：Godot 插件 + 独立 server 组合，更偏“在 Godot 编辑器内双向读写工程”。
    - 适合场景：当后续需要直接从 AI 修改场景树、脚本和资源时，可进一步评估。
