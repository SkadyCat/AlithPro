# 交付说明：suggestions-2026-04-04T14-41-12-315Z

## 源任务文件
`suggestions-2026-04-04T14-41-12-315Z.md`

## 任务目标
TileView 控件应支持指定 `entry-class`（对应 UE5 的 EntryWidgetClass），并创建对应的独立 Entry 控件。

## 所做改动

### 新建文件
- **`application/elements/UTileEntry.js`** — TileView/ListView 默认条目控件
  - 继承 UWidget，注册为 `<umg-tile-entry>`
  - `setListItem(item, index)` 方法（等价 UE5 OnListItemObjectSet）
  - `getListItem()` / `getListIndex()` 获取绑定数据
  - `setSelected(bool)` / `isSelected` 选中状态
  - `onListItemObjectSet()` 子类可重写自定义渲染
  - 默认渲染：icon + label + subtitle + qty

### 修改文件
- **`application/elements/UTileView.js`** — 增强 entry-class 支持
  - 新增 `entry-class` 属性（observedAttributes）
  - 新增 `EntryWidgetClass` getter
  - `_renderTiles()` 分支：有 entry-class 时创建自定义元素实例，否则回退到内联 HTML
  - 新增 `getEntryInstances()` / `getEntryAt(idx)` API
  - 新增 `_attachEntryEvents()` / `_attachTileEvents()` 拆分事件绑定
  - `_syncSelection()` 支持 entry 模式调用 `setSelected()`
  - 完全向后兼容：不设 entry-class 时行为不变

- **`application/elements/index.js`** — 新增 `export { UTileEntry }`

- **`application/public/panels/bag-panel.js`** — TileView 添加 `entry-class="umg-tile-entry"`

- **`application/public/editor.html`** — inspector TileView 属性面板新增 EntryWidgetClass、EntryInstances 行

## 验证结果
- `node --check` 通过：UTileEntry.js、UTileView.js、index.js
- `http://127.0.0.1:3200/editor.html` HTTP 200

## 剩余风险
- bag-panel 的 `renderGrid()` 仍使用自定义 template 函数（非 entry-class 模式的 onListItemObjectSet），因为需要内联 selected 样式；若需完全切换到 entry 模式需调整 renderGrid 逻辑
