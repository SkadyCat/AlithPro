任务目标：
- 让多风格提案不只停留在颜色层，而是加入布局层、信息层次层的实际变化。

所做改动：
- 更新 `application\public\index.html`：
  - 为背包主区域增加 `inventory-stage` 与 `layout-banner`，专门展示当前主题对应的布局方案说明。
  - 调整背包区结构，让详情卡与格子区可以在不同主题下交换顺序或改变主次关系。
- 更新 `application\public\styles.css`：
  - 新增 `data-layout` 驱动的布局系统。
  - 为 4 套主题分别提供布局级差异：
    - `scroll`：卷轴式双栏，详情右侧固定。
    - `companion`：陪伴式偏置布局，详情区左置、格子区右置且更密。
    - `menu`：主机菜单式横向展开，先看格子区，详情卡下沉到第二层。
    - `dashboard`：中台面板式工作区，头部和主工作区分层更明显。
- 更新 `application\public\app.js`：
  - 为每个主题增加 `layout`、`layoutName`、`layoutDescription` 元数据。
  - 切换主题时同步更新 `body.dataset.layout`，使页面结构和文案说明一起变化。

效果结论：
- 现在不同主题之间不再只是“换色皮肤”，而是同时改变：
  - 头部信息编排
  - 详情区与背包区的位置关系
  - 背包格子密度
  - 主次阅读顺序
- 这样挑风格时，能更真实地比较“这个方向适不适合题材”，而不只是比较配色。

验证结果：
- `npm run check` 通过。
- 访问 `http://127.0.0.1:3200/` 可返回包含 `layout-banner`、`inventory-stage` 的新版页面 HTML。

剩余风险或阻塞项：
- 当前布局差异仍是静态结构切换，尚未引入主题专属交互组件（如主机式标签导航、二次元角色立绘栏、国风卷轴分页等）；若继续深挖，需要针对选中的方向再做一轮定制组件。
