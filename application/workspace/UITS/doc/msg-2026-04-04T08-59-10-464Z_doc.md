# 交付说明：修复页签控件缺少图片

## 源任务文件
`msg-2026-04-04T08-59-10-464Z.md`

## 目标
调查并修复 TabEquipment 等控件在生成的场景中没有图片的问题。

## 根因分析
`generate_bag_ui.gd` 在构建 .tscn 时只设了页签的文本和尺寸，没有写入图标和背景样式。这些视觉属性仅由 `main.gd` 在运行时加载，导致在 Godot 编辑器中打开场景时页签、关闭按钮、格子均为裸白按钮。

## 改动内容

### generate_bag_ui.gd
- **新增常量**：`TAB_ICON_PATHS`（4 个页签图标路径）、`TAB_IDLE_TEXTURE`、`TAB_ACTIVE_TEXTURE`、`ITEM_CARD_TEXTURE`。
- **页签按钮**：写入 icon、alignment、icon_alignment、font_size、h_separation 以及 normal/hover/pressed/focus 四态 StyleBoxTexture。
- **关闭按钮**：写入 font_size 和四态 StyleBoxTexture。
- **物品格子**：写入 normal/hover/pressed 三态 StyleBoxTexture（使用 item_card_bg）。
- **新增辅助函数** `_make_stylebox(texture_path, margin)` 用于构建 StyleBoxTexture。

## 验证结果
- `npm run generate:bag-godot-ui` 成功，Godot 4.6.2 无错误。
- 生成的 bag_ui.tscn 现在包含页签图标、页签背景、关闭按钮背景、格子卡片背景。

## 风险与阻塞
- `main.gd` 在运行时仍会重新应用完整主题，覆盖生成时写入的样式，因此不存在冲突风险。
