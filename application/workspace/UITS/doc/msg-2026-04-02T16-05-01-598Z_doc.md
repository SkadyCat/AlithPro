任务目标：
- 集成一个类似 F12 的页面检视功能，可在当前演示页中选择控件并看到节点树。

所做改动：
- 更新 `application\public\index.html`：
  - 新增 `F12 检视器` 悬浮按钮。
  - 新增右侧 `inspector-panel` 侧边栏，用于显示当前选中节点与节点树。
  - 新增 `inspector-overlay` 高亮框，用于标记当前选中的控件。
- 更新 `application\public\styles.css`：
  - 为检视器按钮、侧边栏、节点树、选中态和高亮覆盖层添加样式。
  - 在检视器打开时为页面右侧留出空间，避免主内容被完全遮挡。
- 更新 `application\public\app.js`：
  - 新增 `F12` 键监听，可开关检视器。
  - 新增页面点击选中逻辑：检视器开启后，点击页面中的控件即可选中。
  - 新增节点描述、节点树渲染、树节点点击反选和高亮框同步逻辑。
  - 自动过滤检视器自身节点，避免把侧边栏和按钮本身混入主页面节点树。

使用方式：
- 打开页面后按 `F12`，或点击右下角 `F12 检视器` 按钮。
- 检视器开启后：
  - 点击页面控件，可看到右侧“当前节点”信息。
  - 下方节点树会同步高亮当前选中的元素。
  - 点击节点树里的节点，也可以反向选中页面元素。

验证结果：
- `npm run check` 通过。
- 访问 `http://127.0.0.1:3200/` 返回包含 `inspector-panel`、`inspector-overlay`、`F12 检视器` 标记的页面 HTML。

剩余风险或阻塞项：
- 当前节点树基于运行时 DOM 结构，适合调试和观察；若后续需要“像设计器一样”拖拽、重排、修改属性，还需要继续扩展成真正的可编辑树和属性面板。
