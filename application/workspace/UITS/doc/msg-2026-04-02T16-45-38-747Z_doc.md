任务目标：
- 不再凭空猜“游戏 UI 应该长什么样”，而是参考网络上的优秀案例去改。
- 提升当前预览工作台的实际观感，让 `center_area` 更像可玩的游戏界面。

参考结论：
- 参考来源：
  - Justinmind 关于 game UI design 的公开文章：强调游戏 UI 要兼顾沉浸感、信息清晰度和玩家操作效率。
  - Mockplus 相关页面未直接返回有效案例内容，但结合前一来源，确认高质量游戏 UI 的重点不是花哨，而是“贴题材 + 清晰层级 + 关键操作显眼”。
- 提炼出的可执行原则：
  - 视觉要服务题材，而不是堆装饰。
  - 关键信息必须第一眼能读到：资源、任务、角色状态、可操作按钮。
  - 操作入口要比背景装饰更显眼，不能只剩一堆漂亮方块。

所做改动：
- 更新 `application\public\styles.css`：
  - 进一步收敛工作台外壳，让外部结构更克制、更像工具界面。
  - 加强菜单激活态、canvas 面板边界和主题一致性。
- 更新 `application\public\app.js`：
  - 为 `center_area` 增加 `drawText()` 与 `drawStrokeText()`，让预览中出现真实的 UI 信息，而非只有色块。
  - 升级三个场景：
    - `主 HUD`：增加等级、HP、金币、钻石、时间、Boss 计时、任务追踪、技能名称。
    - `背包界面`：增加分类按钮、容量、道具名、品质、耐久、属性、推荐搭配与操作按钮。
    - `剧情对话`：增加角色名、身份、对话文本、AUTO/LOG 标签与可选分支按钮。

效果结论：
- 现在 `center_area` 更接近“真实游戏截图式预览”，而不是一组概念草图。
- 预览里已经能看到：状态、资源、目标、说明、行动按钮这些真正玩家会用到的东西。
