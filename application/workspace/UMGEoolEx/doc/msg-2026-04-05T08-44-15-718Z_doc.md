# 交付说明 — 引用蓝图级联展开

## 源任务
`msg-2026-04-05T08-44-15-718Z.md`

## 目标
引用的 EntryClass 和子蓝图在下一列展开，依次递归。

## 改动

### uasset-parser.js
- 新增 `findBlueprintRef()` 函数：检测 export 的 classIndex 是否指向 `WidgetBlueprintGeneratedClass` 或 `BlueprintGeneratedClass` 的 import，并追溯其 outer Package 获取 `/Game/` 路径
- `buildNodeTree()` 中为引用了其他蓝图的节点添加 `blueprintRef` 字段
- `parseUAsset()` 返回值新增 `blueprintRefs` 数组：扫描 import 表中所有 WidgetBlueprintGeneratedClass/BlueprintGeneratedClass 的跨蓝图引用

### server.js
- 新增 `GET /api/resolve-path?path=/Game/...` 端点：将 `/Game/` 相对路径转换为磁盘上的 `.uasset` 绝对路径

### index.html
- 中心区域从单树改为 `#columns-container` 多列容器

### style.css
- 新增 `.node-column`、`.column-header`、`.nt-ref-icon`、`.has-ref` 样式
- 列容器支持水平滚动

### app.js
- 新增多列管理：`clearColumns()`、`addColumn()`、`removeColumnsFrom()`、`addTreeColumn()`
- 新增 `openBlueprintRef()`：解析引用路径 → 加载蓝图 → 在下一列渲染 widget tree
- 每列底部显示「引用蓝图」链接区（从 `blueprintRefs` 提取，排除自引用）
- 点击引用或带 `→` 标记的节点即可在下一列展开子蓝图，清除右侧旧列
- 递归支持：每一列都能继续产生新列

## 验证
- 语法检查通过
- BagPanel 正确检测出 1 个引用 `WBP_BagItemEntry_C` → `/Game/UI/Panels/WBP_BagItemEntry`
- `resolve-path` 正确将 `/Game/` 路径转换为磁盘路径
- BagItemEntry 解析成功（10 exports），无进一步引用（递归终止）
- 服务器 HTTP 200

## 风险
- 蓝图间引用通过 import 表检测；通过属性数据（如 ListView.EntryWidgetClass）间接引用的也已覆盖
- 不在 `/Game/` 下的引擎类引用已自动排除
