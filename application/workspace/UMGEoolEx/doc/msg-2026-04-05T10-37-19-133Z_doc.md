# 交付说明：蓝图数据结构 JSON 生成

## 源任务文件
`msg-2026-04-05T10-37-19-133Z.md`

## 目标
添加按钮，为蓝图生成对应的 JSON 数据结构。控件类型映射到 UE 数据类型（TileView/ListView→TArray, TextBlock→FText, Image→UTexture2D 等），生成的 JSON 可用于通过序列化方式驱动 UI 显示。

## 改动内容

### app.js
- `renderAssetInfo()` 新增「生成数据结构」按钮（绿色主题，与已有 Export JSON 并排）
- 新增 `WIDGET_TYPE_MAP` 常量：定义 13 种 UMG 控件到 UE 数据类型的映射
  - TileView/ListView/TreeView → `TArray<UObject*>`
  - TextBlock/EditableText/RichTextBlock → `FText`
  - Image → `UTexture2D*`（附带当前贴图引用作为 defaultValue）
  - Button/CheckBox → `bool`
  - ProgressBar/Slider/SpinBox → `float`
  - ComboBoxString → `FString`
- 新增 `generateDataSchema(fileName, roots)` 函数：遍历 widget tree，收集所有可数据化控件
- 新增 `showSchemaPanel(schema, fileName)` 函数：
  - 右侧面板显示数据结构摘要表格 + JSON 预览
  - 提供「复制 JSON」和「下载」两个操作按钮

### style.css
- 新增 `.schema-json` 样式（暗色代码块，带滚动）
- 新增 `.schema-btn` 按钮样式（hover 高亮）

## 输出 JSON 格式示例
```json
{
  "blueprint": "PanelPlayerEquip",
  "description": "PanelPlayerEquip 界面数据结构 - 通过 JSON 序列化驱动 UI 显示",
  "fields": [
    { "widgetName": "HeroIm", "widgetClass": "Image", "dataType": "UTexture2D*", "defaultValue": "/Game/UI/.../HeroExampleImage" },
    { "widgetName": "TextBlock_112", "widgetClass": "TextBlock", "dataType": "FText" },
    { "widgetName": "left_lst", "widgetClass": "TileView", "dataType": "TArray<UObject*>" }
  ]
}
```

## 验证结果
- PanelPlayerEquip 生成 10 个字段：1 Button(bool), 5 Image(UTexture2D*), 2 TextBlock(FText), 2 TileView(TArray)
- Image 字段均附带 defaultValue 贴图路径
- 语法检查通过，服务器正常运行

## 风险 / 阻塞
- 控件类型映射基于已知 UMG 类名，自定义 C++ 控件不会被自动识别
- 当前仅扫描 WidgetTree 子节点，不包含 Blueprint 自身变量
