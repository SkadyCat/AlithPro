# 交付说明 - BasePanel 转化修复

## 源任务
`msg-2026-04-05T08-09-41-891Z.md`

## 目标
修复 BasePanel.uasset 转化/解析失败问题。

## 问题分析
BasePanel.uasset 包含两个 WidgetTree 导出（一个在 WidgetBlueprint 下，一个在 WidgetBlueprintGeneratedClass 下），导致前端 `extractWidgetTree` 提取到重复的 CanvasPanel_35 节点。

此外，classIndex 指向另一个 export（而非 import）时，显示为 `Export_10` 而非正确的类名 `BasePanel_C`。

## 改动

### `lib/uasset-parser.js`
- `getClassName()` 新增 `exports` 参数，支持解析 export-to-export 的类引用
- `buildNodeTree()` 和返回数据中的 className 均使用更新后的解析逻辑

### `public/app.js`
- 重写 `extractWidgetTree()`：优先提取 `WidgetBlueprint` 下的 WidgetTree 子节点，跳过 `WidgetBlueprintGeneratedClass` 下的 CDO 副本，消除重复渲染

## 验证结果
- `npm run check` 通过
- BasePanel.uasset (12 exports)、WBP_ExamplePanel (29 exports)、WBP_BagPanel (147 exports) 均正常解析
- 前端只显示主 WidgetTree 下的节点，无重复
