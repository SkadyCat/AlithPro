# 交付说明：贴图预览 - 显示实际 PNG 图片

## 源任务文件
`msg-2026-04-05T09-52-12-678Z.md`

## 目标
选中 Image 节点时，在右侧面板直接展示引用贴图的实际图片（PNG），而非仅显示路径。

## 改动内容

### server.js
- 新增 `GET /api/texture-preview?path=/Game/...` 端点
  - 读取 Texture2D 的 `.uasset` 文件，扫描内嵌 PNG 数据（通过 PNG magic header 和 IEND 标记定位）
  - 返回最大尺寸的 PNG（即全分辨率 mip），Content-Type 为 `image/png`
  - 支持 `/Game/` 路径和绝对路径两种输入
  - 设置 1 小时缓存头减少重复读取

### app.js
- `showNodeDetail()` 中，当 `ref.class === 'Texture2D'` 时，在引用资源下方渲染 `<img>` 标签
  - 图片 src 指向 `/api/texture-preview` 端点
  - 含 `onerror` 降级处理，显示"预览不可用"

### style.css
- 新增 `.texture-preview-wrap` 和 `.texture-preview` 样式
  - 暗色背景框、圆角、最大高度 260px 自适应

## 技术发现
- UE5 Texture2D `.uasset` 文件内嵌完整 PNG 图片数据（通常 2 张：低分辨率 mip + 全分辨率）
- PNG 数据通过标准 magic bytes (`89 50 4E 47`) 和 IEND chunk 标记即可可靠定位
- 无需 DXT/BC 解压，直接提取 PNG 二进制即可

## 验证结果
- EquipmentFrame2: 200 OK, image/png, 531KB (662×1024)
- HeroExampleImage: 200 OK, image/png, 106KB (512×822)
- 不存在的资源: 404 正确返回
- 主页面正常加载

## 风险 / 阻塞
- 少数 UE5 贴图可能使用非 PNG 内嵌格式（如纯 DXT bulk data），此时端点返回 404
- 大尺寸贴图（如 4K）的 PNG 可能较大，但有缓存头缓解
