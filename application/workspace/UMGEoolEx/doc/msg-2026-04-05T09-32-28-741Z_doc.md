# 交付说明：UImage 选中时显示引用的 Image/Texture

## 源任务文件
`msg-2026-04-05T09-32-28-741Z.md`

## 目标
选中 UImage（及其他控件）时，在右侧信息面板显示该节点引用的贴图、材质等资源。

## 改动内容

### uasset-parser.js
- 新增 `scanExportForImportRefs(buf, exp, imports, nameTable)` 函数
  - 逐字节扫描 export 序列化数据，匹配 Texture2D / Material / Font / SoundWave 等目标 import 索引
  - 解决了 UE5 tagged property 字段对齐偏移导致常规属性解析失败的问题
- `buildNodeTree()` 增加 `buf` 参数，为每个 tree node 附加 `resourceRefs` 数组
- `parseUAsset()` 将 buf 传递给 buildNodeTree

### server.js
- `/api/parse` 无需改动，tree 返回值已自动包含 resourceRefs

### app.js (前端)
- `showNodeDetail()` 新增"引用资源"展示区域，显示每条 ref 的 class、name、packagePath

## 验证结果
- PanelPlayerEquip 的 5 个 Image 节点均正确映射到各自的 Texture2D（Bg2→EquipmentFrame2, HeroIm→HeroExampleImage 等）
- Button 节点也正确显示 Normal/Pressed 两个贴图引用
- API 返回数据完整，前端可正常渲染

## 风险 / 阻塞
- 字节扫描在极大 export 上理论上可能产生误匹配，但实测未观察到 false positive
- 暂不支持 SoftObjectPath 类型的资源引用（仅扫描硬引用 import index）
