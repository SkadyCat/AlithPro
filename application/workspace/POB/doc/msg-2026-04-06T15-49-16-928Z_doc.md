# 交付说明: 修复分解弹窗子弹窗定位

## 源任务
`msg-2026-04-06T15-49-16-928Z.md` — 点击装备增伤弹窗内的词条后，弹出的装备弹窗应出现在右侧而非左上角

## 根因
点击 `.decomp-item-link` 时先执行 `popup.remove()` 移除当前弹窗，然后调用 `showItemPopup(row, true)`。但 `row` 已从 DOM 移除，`getBoundingClientRect()` 返回全零，导致弹窗定位在 (0, 0)。

## 修复

### `application/public/js/app.js`
1. `showDecompPopup()` 中：点击行之前先用 `popup.getBoundingClientRect()` 捕获当前弹窗位置，构造 `anchorRect` 锚定在弹窗右侧
2. `showItemPopup(badge, pinned, anchorRect)` 新增第三个可选参数 `anchorRect`：
   - 有 `anchorRect` 时：子弹窗左边缘紧贴父弹窗右边缘 + 6px，顶部对齐
   - 无 `anchorRect` 时：保持原有行为（badge 下方）
   - 增加 `top < 10` 边界保护

## 验证
- 语法检查通过，服务器正常重启
- 子弹窗将定位在分解弹窗右侧（而非 0,0）

## 风险
无
