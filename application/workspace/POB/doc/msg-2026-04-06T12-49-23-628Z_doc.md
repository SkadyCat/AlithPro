# 交付说明: 修复词条勾选切换不触发重新计算

## 源任务
`msg-2026-04-06T12-49-23-628Z.md`

## 目标
用户反馈取消勾选 "276% increased Physical Damage" 后伤害计算没有变化。

## 根因分析
1. **`escHtml()` 不安全**: `escHtml(s)` 直接调用 `s.replace()`，若 `s` 为 `null`/`undefined` 则抛异常
2. **错误被静默吞掉**: `refreshDamageBreakdown()` 的 `catch { /* silent */ }` 把渲染异常全部吞掉
3. **无视觉反馈**: DPS 对比条在顶部、计算链在底部，用户可能看不到变化；没有加载提示

## 改动
- `escHtml()`: 改为 `String(s ?? '').replace(...)`, null-safe
- `refreshDamageBreakdown()`:
  - 移除静默 catch，改为 `console.error` 输出
  - 增加 "⏳ 重新计算中…" 加载横幅
  - 失败时显示 "❌ 计算失败" 提示
- `toggleMod()`: 增加 `console.log` 调试输出
- `renderDamageBreakdown()`: "总 DPS" 计算链步骤增加内联估算值 (`➜ 估算: ...`)
- `style.css`:
  - DPS 对比条增加 `position: sticky; top: 0` 保持可见
  - `.dps-loading` 加载提示样式
  - `.chain-step-changed` 变化步骤高亮
  - `.chain-estimated` 内联估算值样式

## 验证
- API 切换确认: 禁用 276% → DPS 从 8.76M 降至 3.05M ✅
- 武器物理: 469-1056 → 125-281 ✅
- JS 语法检查通过 ✅
- 所有新增 CSS 类和 JS 逻辑已部署到服务器 ✅

## 剩余风险
- 浏览器端实际交互测试需用户确认
- 如有其他渲染字段为 null 的情况，现在 escHtml 已能安全处理
