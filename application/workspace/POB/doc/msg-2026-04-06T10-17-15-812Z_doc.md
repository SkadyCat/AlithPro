# 交付说明 — 修复装备显示 (名称/稀有度/格式)

## 源任务
`msg-2026-04-06T10-17-15-812Z.md` — 用户对比了我们工具与 POB 的装备显示差异

## 目标
修复装备显示为 "未知物品 (NORMAL)" 的问题，正确解析并展示物品名称、稀有度、基础类型。

## 根因
POB XML 的 `<Item>` 元素中 `@_rarity` 和 `@_name` 属性**通常为空**，真实数据存储在 `#text`（物品文本体）中：
- 第 0 行: `Rarity: RARE/UNIQUE/MAGIC/NORMAL`
- 第 1 行: 物品名称
- 第 2 行: 基础类型 (仅 RARE/UNIQUE)

旧代码 `build-codec.js` 直接读取 XML 属性，回退为 `'NORMAL'` / `''`，导致所有物品显示为 "未知物品 (NORMAL)"。

## 改动

### 1. `application/src/core/build-codec.js` — normalizeItems()
- 从 `#text` 解析 rarity、name、baseType
- MAGIC 物品: 只有 1 行名称，不误读 `Unique ID:` 为基础类型
- 格式: RARE/UNIQUE → "名称 (基础类型)", MAGIC → 仅名称
- 保留 XML 属性优先级 (有 `@_name` 就用它)
- 同时从文本解析 Quality 和 Corrupted 状态

### 2. `application/public/js/app.js` — renderItems() 重写
- 按装备栏顺序显示 (武器→护甲→饰品→药剂→珠宝)
- 新增 `renderItemCard()`: 分离 slot/rarity 标签 + 彩色名称
- 新增 `formatItemMods()`: 分类显示词缀 (explicit/crafted/fractured/implicit/corrupted)
- 跳过元数据行 (Rarity/Unique ID/Sockets 等)

### 3. `application/public/css/style.css` — 装备卡片样式
- 按稀有度添加边框颜色 (unique=棕, rare=黄, magic=蓝)
- 词缀分类上色: crafted=浅蓝斜体, fractured=紫, corrupted=红色粗体
