# 交付说明 — 伤害计算链词条归因与开关

## 源任务
`msg-2026-04-06T12-08-50-781Z.md`

## 目标
在伤害计算链中显示具体是哪个装备/宝石/道具的哪个词条贡献了伤害，并允许通过 checkbox 临时取消词条后重新计算。

## 改动

### `application/src/analysis/damage-breakdown.js` — 全面重写
- 新增 `collectAllMods(build)` — 从所有装备/珠宝/药剂中提取伤害相关词条，每条分配唯一 ID
- 新增 `collectGemMods(group)` — 从辅助宝石中提取 more/less/crit 条目
- 新增 `parseDamageMod()` — 解析单条词条并分类（17 种类别）
- 新增 `computeWeaponLocal()` — 仅使用已启用词条计算武器伤害
- 新增 `computeFlatBase()` — 仅使用已启用词条计算总基础
- 新增 `buildAttributedChain()` — 每个链步骤附带 `contributors` 数组标识来源词条
- 入口 `analyzeDamageBreakdown(build, disabledModIds=[])` 接受禁用列表
- DPS 估算逻辑：用原始全词条的 flat base 推算 globalMult，再用调整后的 flat base 估算新 DPS

### `application/src/routes/build.js`
- `/api/build/damage-breakdown` 端点接受可选 `disabledMods` 数组参数

### `application/public/js/app.js`
- 新增 `disabledMods` Set 和 `currentBuildCode` 状态
- 重写 `renderDamageBreakdown()` — 词条按来源分组，每条带 checkbox
- 新增 `toggleMod()` — checkbox 变化时 300ms 防抖后调用 API 重算
- 新增 `refreshDamageBreakdown()` — 发送 disabledMods 到后端
- 新增 `resetAllMods()` — 一键恢复全部词条
- DPS 对比条：禁用词条后显示原始→估算 DPS 及变化百分比
- 链步骤带来源徽章 (contributor badges)
- 辅助函数：`getSlotIcon()` 槽位图标、`modCategoryLabel()` 类别中文标签

### `application/public/css/style.css`
- 新增：`.mod-group`, `.mod-entry`, `.mod-disabled` 词条列表样式
- 新增：`.dps-compare`, `.dps-down`, `.dps-up` DPS 对比条样式
- 新增：`.contrib-badge`, `.contrib-disabled` 来源徽章样式

## 验证
- 语法检查通过（server.js, damage-breakdown.js, build.js）
- API 测试：Deadeye Lv.100 build 解析出 30 个词条，17 步计算链
- 禁用 Quality(+30%) + 276% inc phys → 武器物理从 469-1056 降至 96-216
- DPS 估算：8,760,691 → 2,571,571（-70.6%）符合预期
- 无禁用词条时 estimatedDPS = originalDPS ✓

## 限制与风险
- DPS 估算仅精确反映武器局部和装备附加伤害的变化
- 全局增伤/more/暴击词条的开关目前不影响 DPS 估算（因 global multiplier 来自 POB 不可分解）
- 后续可通过追加全局增伤比例估算进一步改善精度
