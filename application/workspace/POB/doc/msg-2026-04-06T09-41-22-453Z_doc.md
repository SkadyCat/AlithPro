# 交付说明 — 修复武器基础伤害解析错误

## 源任务
`msg-2026-04-06T09-41-22-453Z.md`

## 问题
武器伤害分析显示"物理 0.0 + 元素 79.5"，因为解析器只读取了物品文本中的词缀（如"Adds 14 to 197 Lightning"），没有考虑武器基底类型（如 Kinetic Wand）自带的固有物理伤害。

## 根因
POB 的物品 XML 不存储武器基底面板数据——只存储物品类型名称和词缀文本。基底伤害（PhysicalMin/Max、CritChanceBase、AttackRateBase）来自游戏数据文件。

## 修复内容

### 新建文件
- **`application/src/data/weapon-bases.json`** — 362 种武器基底类型数据库，从 POB `Data/Bases/*.lua` 提取
  - 每条记录：`{ physMin, physMax, critBase, atkRate, type }`

### 修改文件
- **`application/src/analysis/damage-breakdown.js`**
  - 加载 `weapon-bases.json` 查找基底类型
  - 武器物理伤害 = 基底 × (1+品质/100) × (1+局部增物理%/100)
  - 武器攻速 = 基底 × (1+局部增攻速%/100)
  - 伤害链从4步扩展为11步：基底→品质→局部词缀→武器面板DPS→全局加成→最终DPS

## 修复前后对比（Kinetic Wand, Q20, 18%攻速）
| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| 物理平均 | 0.0 | 49.8 |
| 元素平均 | 105.5 | 105.5 |
| 总平均 | 105.5 | 155.3 |
| 武器攻速 | (无) | 1.89 |
| 武器面板 DPS | (无) | 293.5 |

## 验证结果
✅ Kinetic Wand 基底查找正确（physMin=29, physMax=54）
✅ 品质和局部增伤正确应用
✅ 11步完整伤害链输出
