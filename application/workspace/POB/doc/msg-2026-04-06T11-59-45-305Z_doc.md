# 交付说明 — 左侧栏多 Build 管理

## 源任务
`msg-2026-04-06T11-59-45-305Z.md`

## 目标
在 POB Node 网页左侧添加固定侧栏，展示所有已保存的 Build，支持点击切换、新建导入和删除，取消原来的弹窗式保存/加载。

## 改动

### `application/public/index.html`
- 新增 `.app-layout` 网格布局（260px 侧栏 + 主内容区）
- 侧栏包含：标题 + 新建按钮、搜索框、`#buildList` 容器
- 移除旧的 `btnSave` / `btnLoad` 按钮
- 主内容区 header 增加 `#headerTitle` / `#headerSubtitle` 动态标题

### `application/public/css/style.css`
- `.app-layout` grid、`.sidebar`、`.build-list`、`.build-entry`、`.be-*` 样式
- 激活态 `.build-entry.active` 高亮
- 响应式 ≤768px 隐藏侧栏

### `application/public/js/app.js`
- 新增 `currentFilename` / `buildListCache` 状态
- `refreshBuildList()` — 从 `/api/session/list` 拉取列表并渲染
- `selectBuild()` — 点击切换 Build
- `deleteBuild()` — 调用 DELETE 接口删除
- `autoSave()` — 解码后自动保存并刷新列表
- `updateHeader()` — 动态更新标题
- `showImportNew()` / `clearBuild()` — 新建/清空
- 搜索过滤、职业图标
- 移除旧 `saveSession()` / `loadSession()` 弹窗逻辑

### `application/src/routes/session.js`
- 新增 `DELETE /api/session/delete/:filename` 端点
- `/api/session/list` 返回中增加 `dps` 字段

## 验证
- 语法检查 `node --check server.js` + `session.js` 通过
- 服务器重启成功，`/api/session/list` 返回 2 个已保存 Build
- 所有 HTML 元素 ID 与 JS 选择器对齐，无 `btnSave`/`btnLoad` 残留引用

## 风险
- 无自动去重机制，每次导入同一 Build Code 会创建新文件
- 侧栏仅在宽屏显示，移动端需后续处理
