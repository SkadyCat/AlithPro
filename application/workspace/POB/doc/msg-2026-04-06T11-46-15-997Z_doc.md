# 交付说明 — 修复武器物理伤害公式

## 源任务
`msg-2026-04-06T11-46-15-997Z.md` — 用户反馈 POB 显示物理 469-1056，我们工具计算不对

## 目标
修正武器本地物理伤害公式，使结果与 POB 完全一致。

## 根因
通过阅读 POB 源码 `Classes/Item.lua:1457-1458`，确认 POB 使用的公式是**乘法**模式：

```lua
min = round(min * (1 + physInc / 100) * (1 + qualityScalar / 100))
max = round(max * (1 + physInc / 100) * (1 + qualityScalar / 100))
```

即 `(基础 + 附加) × (1 + 物理增加%/100) × (1 + 品质/100)`。
之前 Task 9 错误地改为加法 `(1 + (品质 + 物理增加%)/100)`。

同时确认：`8% increased Explicit Physical Modifier magnitudes` 在当前版本 POB 中未实现（ModCache 产出空列表），不影响计算。

## 改动

### `application/src/analysis/damage-breakdown.js`
1. **修正公式**：`Math.round((base + flat) × (1 + incPhys/100) × (1 + quality/100))` — 乘法模式 + round
2. **攻击速度累加**：`attackSpeedInc` 从覆盖改为累加（支持隐性+显性同时存在）
3. **攻击速率精度**：`Math.round(base × (1+AS/100) × 100) / 100` — 2位小数，匹配 POB

## 验证
- **Spine Bow (Deadeye)**：物理 469-1056 ✅（与 POB 完全一致）
  - base 38-115, flat 58-101, inc 276%, quality 30%
  - 96 × 3.76 × 1.30 = 469, 216 × 3.76 × 1.30 = 1056
- **Kinetic Wand (力爆-day1)**：物理 35-65, 攻速 1.89
  - base 29-54, flat 0, inc 0%, quality 20%

## 剩余风险
