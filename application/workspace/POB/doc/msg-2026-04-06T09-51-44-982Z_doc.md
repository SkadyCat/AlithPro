# 交付说明 — 修正武器物理伤害计算逻辑

## 源任务
`msg-2026-04-06T09-51-44-982Z.md`

## 问题
用户指出武器物理伤害应为 469~1056，但工具显示的是武器局部物理（如 34.8-64.8）。

## 分析

### 两层差异
1. **公式 Bug（已修复）**：品质和局部增物理%应该是**加法**关系，而非乘法。  
   - 错误：`base × (1+Q/100) × (1+inc/100)`  
   - 正确：`(base + flat_added) × (1 + (Q + inc) / 100)`
   - 影响：对有局部%增物理词缀的武器（如 Dire Goad +45%）结果偏差较大

2. **局部 vs 全局**：469~1056 是 POB 完整计算引擎的每击物理伤害（含天赋树、辅助宝石、光环等全部加成），非武器局部面板。工具现在的武器局部物理（34.8-64.8）数值正确，但需要加上全局增幅才能得到每击伤害。

### 全局增幅无法精确复现
POB 的 CalcOffence.lua（340KB）包含完整的伤害计算：各类型分别计算、转换、穿透、天赋树数百个节点。当前 Node 工具不含此引擎，只能：
- 收集全部装备的平坦附加伤害（已实现）
- 从 AverageDamage 反推总倍率（已实现）
- 按平坦基数比例估算各类型每击伤害（近似，因为物理和元素的增幅倍率不同）

## 修复内容

### `application/src/analysis/damage-breakdown.js`

1. **物理伤害公式修正** — `(base + flat) × (1 + (Q + incPhys%) / 100)`
2. **新增 `computeTotalFlatBase()`** — 汇总武器局部 + 全部装备附加伤害，按类型分类
3. **伤害链扩展为 14 步** — 新增：装备附加、总平坦基础、全局倍率（反推）、各类型估算单击
4. **标签更新** — "武器局部物理" / "武器局部元素" 明确标注为局部值
5. **暴击前/后拆分** — 平均伤害拆分为暴击前均值和含暴击均值

### `analyzeDamageBreakdown()` 返回新增字段
- `totalFlatBase` — 所有平坦伤害来源汇总（byType + totalMin/Max/Avg）
- `damageChain.perTypeEstimate` — 各类型估算每击（近似值）

## 验证结果
- ✅ Blight Scratch (Kinetic Wand, Q20, 无增物理): 34.8-64.8 → 不变
- ✅ Dire Goad (Kinetic Wand, Q20, +45%增物理, +21-45物理): 82.5-163.4 → 公式修正后
  - 旧结果（错误）：71.5-139.0
  - 新结果（正确）：82.5-163.4 = (29+21)×1.65 / (54+45)×1.65
- ✅ 装备附加伤害正确收集
- ✅ 全局倍率反推显示
- ✅ 14步完整伤害链输出

## 残留风险
- 各类型估算每击是**近似值**（假设统一倍率），实际物理和元素的增幅系数不同（元素流 build 中物理每击会比估算低很多）
- 精确的各类型每击需要复现 POB 的 CalcOffence 引擎（约 340KB Lua 代码），当前未实现
