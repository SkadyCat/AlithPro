# 交付说明 — 修复 POB 构建代码解码失败

## 源任务
`msg-2026-04-06T08-53-37-547Z.md`

## 目标
修复特定 13,936 字符 POB 构建代码无法解码的问题。

## 问题根因
POB 构建代码使用**标准 zlib 压缩**（带 `0x78` 头部），而非原始 deflate（无头部）。
原始实现中 `build-codec.js` 使用 `zlib.inflateRawSync()` 只能处理无头部的 raw deflate 数据流。

对于带有 `78 da` / `78 9c` zlib 头的代码（这是 POB 标准格式），该方法会抛出错误。

## 修复内容
- **`application/src/core/build-codec.js`**
  - 解码：检测首字节 `0x78` → 使用 `zlib.inflateSync()`（标准 zlib），否则 fallback 到 `zlib.inflateRawSync()`（兼容 raw deflate）
  - 编码：从 `zlib.deflateRawSync()` 改为 `zlib.deflateSync()`，输出标准 zlib 格式

## 验证结果
| 测试用例 | 结果 |
|----------|------|
| 用户提供的 13,936 字符代码 (Ranger/Deadeye Lv.100) | ✅ 解码成功，DPS=8,760,690 |
| Necromancer Lv.93 build (raw deflate 格式) | ✅ 向后兼容 |
| `/api/build/calculate` 全链路 | ✅ 解码+计算正常 |

## 残留风险
- 无。双模式检测确保标准 zlib 和 raw deflate 均可处理。
