## 结构分析结果

- 应用根目录以 `server.js` 为唯一后端入口，`package.json` 仅提供 `npm start -> node server.js`，依赖只有 `express`。
- `server.js` 同时负责静态资源托管和 JSON 文件 API：文档树 CRUD、图片列表、元素定义、session/uidata/theme 持久化，以及转发到 Alice 的代理接口。
- 前端由 `public\index.html` 加载 `style.css` 与单体脚本 `public\app.js`；没有独立 `src/`、构建步骤或前端打包工具。
- `public\app.js` 是核心实现，包含全局编辑器状态（如 `boxes`、`selectedId`、`zoom`、撤销/重做栈）、画布渲染、控件面板、属性编辑、文档面板、资源面板、预览模式与聊天面板。
- 启动顺序为：页面加载后调用 `loadElements()` 读取控件定义，再 `loadSession()` 恢复会话，随后 `loadTheme('default')` 应用主题，并绑定布局/画布相关观察器与事件。
- 数据持久化以 `application\canvas-editor\data\docs\` 为中心：文档、session、uidata、config 都通过 HTTP API 读写磁盘；主题单独存储在 `data\themes\`。
- `public\bag.html`、根目录下的 `server.log.err`、`server_err_new.txt`、`server_out_new.txt` 属于附属页面或运行产物，不是主启动路径。
