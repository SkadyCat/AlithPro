# 交付说明

- 源任务文件: `msg-2026-03-29T16-44-33-652Z.md`
- 任务目标: 处理 `uieditor` 中出现的 `No \`*.session\` preview data found for this template.` 提示，让预览在缺少 `.session` 时也能工作。

## 所做改动

- 改写了 `application\uieditor\public\app.js` 的预览入口：
  - 新增 `getMetaRoots()`、`defaultWidgetSize()`、`buildMetaPreviewBoxes()`。
  - 当模板缺少 `.session` 数据时，不再直接进入空状态，而是基于 `.meta.json` 的层级结构和属性生成一个近似的 UMG 预览树。
  - `renderSummary()` 现在会把这种情况标记为 `Preview: meta fallback`。
  - 只有在 `.session` 和 `.meta.json` 都无法提供可预览控件时，才显示新的通用空提示。

## 验证结果

- `node --check application\workspace\UIEditor\application\uieditor\public\app.js` 通过。
- 已确认 `public\app.js` 中存在：
  - `buildMetaPreviewBoxes(meta)`
  - `payload.session?.boxes?.length ? payload.session.boxes : buildMetaPreviewBoxes(...)`
  - `Preview: meta fallback`
- 已确认旧的 `No \`*.session\` preview data found for this template.` 单一路径已被替换为回退预览逻辑。

## 剩余风险或阻塞项

- 该回退预览基于 `.meta.json` 合成近似布局，目的是在缺少 `.session` 时避免空白；它不是严格等价于真实 UMG 布局。
- 当前无阻塞项。
