## 具体逻辑分析

### 1. UE 内启动 MCP / 服务桥接

- `Mcp\mcp_server.py` 是 Unreal 编辑器启动脚本，已在 Project Settings → Python → Startup Scripts 中注册。
- Unreal 启动时会自动执行 `start_mcp_server()`，启动 `server_thread.py` 中的 `MCPServerThread`，在后台线程里跑 FastAPI / uvicorn。
- 因 Unreal Python API 不是线程安全的，`mcp_server.py` 通过 `unreal.register_slate_post_tick_callback(_on_tick)` 注册每帧 Tick，把后台线程提交到 `main_thread_queue` 的任务转发回 UE 主线程执行。
- 对应文件：
  - `application\MagicWorld\Mcp\mcp_server.py`
  - `application\MagicWorld\Mcp\server_thread.py`

### 2. 面向外部的 UMG 生成入口

- 真正直接提供“生成 WBP/UMG” HTTP 接口的是 `McpFlask\app.py`，其中 `/ue/create-wbp` 路由负责接收模板名。
- 该接口会把模板名解析成磁盘 JSON 路径：
  - 优先 `G:/GameExPro3/AIUI/umg/{template}.{resolution}.json`
  - 回退 `G:/GameExPro3/AIUI/umg/{template}.json`
- 然后把一个 `_UECallable` 放入 `_ue_log_queue`。这个对象会在 UE 主线程 Slate tick 中执行，内部直接调用：
  - `ue.AIWBPEditorLibrary.create_wbp_from_json(json_path, save_path)`
- 默认生成目标是 `/Game/UI/Panels/WBP_{template}`。
- 对应文件：`application\MagicWorld\McpFlask\app.py`

### 3. 为什么一定要回到 UE 主线程执行

- `McpFlask\app.py` 中 `_UECallable` 的 `__format__()` 会在 UE 主线程日志消费过程中执行真正的 Unreal API 调用。
- 注释已经明确说明：Flask 线程只负责入队，UE 主线程 Slate tick 再消费并执行命令。
- 这样就避免了在 Flask/FastAPI 后台线程里直接调用 Unreal 编辑器 API。

### 4. JSON -> Widget Blueprint 的核心执行器

- `Plugins\AIWBPEditor\Source\AIWBPEditor\Public\AIWBPEditorLibrary.h` 暴露了 Blueprint/Python 可调用接口：
  - `CreateWBPFromJson(JsonFilePath, OutputPackagePath)`
- `AIWBPEditorLibrary.cpp` 的实现非常薄，直接转发给：
  - `FAIWBPImporter::ImportFromJsonToPath(JsonFilePath, OutputPackagePath)`
- 也就是说，真正的 UMG 资产生成逻辑都在 `AIWBPImporter.cpp` 中。
- 对应文件：
  - `application\MagicWorld\Plugins\AIWBPEditor\Source\AIWBPEditor\Public\AIWBPEditorLibrary.h`
  - `application\MagicWorld\Plugins\AIWBPEditor\Source\AIWBPEditor\Private\AIWBPEditorLibrary.cpp`
  - `application\MagicWorld\Plugins\AIWBPEditor\Source\AIWBPEditor\Private\AIWBPImporter.cpp`

### 5. AIWBPImporter 的生成流程

`AIWBPImporter.cpp` 中可以看到完整的“JSON 布局 → UMG 组件树 → Widget Blueprint 资产”流程：

- 读取 JSON 文件并解析节点结构。
- 使用 `WidgetBlueprintFactory` + `AssetTools` 创建新的 `UWidgetBlueprint` 资产。
- 通过 `UWidgetTree` 构建控件树。
- 核心递归函数是 `BuildNode(...)`，按 JSON 节点的 `umg_type` 分派生成不同 UMG 控件。
- 目前明确处理了多类控件，包括：
  - `TextBlock`
  - `Button`
  - `Image`
  - `ProgressBar`
  - `TileView`
  - 以及容器类节点（统一落成 `CanvasPanel` 等结构）
- 几何信息由 `position`、`size`、`anchor` 等 JSON 字段转成 `UCanvasPanelSlot` 的 anchors / offsets / alignment / z-order。
- `ApplySlot`、`ApplySlotFromAnchorJson`、`ApplySlotAuto` 负责把 AIUI JSON 布局坐标映射成 UMG Slot 参数。
- 文本、图片、进度条等控件会进一步读取 `umg_settings` 做细节属性填充。

### 6. TileView / EntryWidget 的特殊处理

- `AIWBPImporter.cpp` 对 `TileView` 有专门分支。
- 注释写明：会读取 `umg_settings` 里的 `entry_width`、`entry_height`、`selection_mode`。
- 同时如果 JSON 中提供 `entry_class_def`，导入器会预先创建对应的 Entry Widget Blueprint，并在构建 `TileView` 时尝试反射设置 `EntryWidgetClass`。
- 文件里还有专门的逻辑“Create a WBP that implements IUserObjectListEntry for use as TileView::EntryWidgetClass”。
- 这说明 MCP 不只是生成一个平面 WBP，而是连 List/Tile 这类需要子条目蓝图的结构都能一起生成。

### 7. 编辑器菜单入口

- 除了 MCP / Flask 远程调用，`AIWBPEditorModule.cpp` 还在 Unreal 编辑器菜单里注册了：
  - `从 UMG JSON 生成 Widget Blueprint`
- 菜单点击后会弹出文件选择框，最终执行同一个核心入口：
  - `FAIWBPImporter::ImportFromJson(...)`
- 说明该能力既可由 HTTP/MCP 远程触发，也可直接在编辑器 UI 中手动触发。

### 8. MagicCore\WBPEditor 的补充作用

- `Plugins\MagicCore\Source\MagicCore\Public\WBPEditor.h` / `Private\WBPEditor.cpp` 提供了一套更底层的 WBP 操作工具：
  - `AddWidgetToBlueprint`
  - `SaveWidgetBlueprint`
  - `AddInterfaceToBlueprint`
  - `SetTileViewEntryClass`
  - `InspectWidgetBlueprint`
- 这些接口可以按蓝图资产路径直接增删控件、添加接口、保存 Widget Blueprint，并可用于 Python 调用。
- 这套能力更偏“对现有 WBP 做编程式编辑/补丁”，而 `AIWBPEditor` 更偏“从 AIUI JSON 一次性生成完整 WBP”。
- 两者组合起来，构成了“先批量生成，再精细修改”的能力基础。

## 端到端链路总结

1. 外部通过 MCP / 本地服务发起请求。
2. `McpFlask\app.py` 的 `/ue/create-wbp` 接口接收模板名，定位 `G:/GameExPro3/AIUI/umg/*.json`。
3. 请求被封装成 `_UECallable` 入队，等待 UE 主线程执行。
4. UE 主线程 Slate tick 消费任务，调用 `unreal.AIWBPEditorLibrary.create_wbp_from_json(...)`。
5. `AIWBPEditorLibrary` 转发到 `FAIWBPImporter::ImportFromJsonToPath(...)`。
6. `AIWBPImporter` 解析 JSON、创建 `UWidgetBlueprint`、用 `BuildNode(...)` 递归构造 `WidgetTree`，设置 Slot、属性、TileView EntryWidget 等。
7. 最终在 UE 内容目录下生成并保存对应的 `WBP_*` 资产，默认路径是 `/Game/UI/Panels`。
