# 留言文档模板优化方案（AI 快速理解版）

> 本文件是对当前留言模板格式的优化建议，供人工审核后再决定是否实施代码改动。

---

## 当前模板结构

```
{空间树dump}

我们目前需要解决的是: {留言内容}
你在解决这个问题之前，你已经具备了以下知识：
{层级知识大纲}

你需要从这个链路去收束你的思考范围。
```

### 当前格式存在的问题

1. **缺少结构化分隔** — 空间树、任务描述、知识大纲连成一片，AI 需要靠自然语言推断边界
2. **文档引用内联** — `需求文档: [...] 交付文档: [...]` 嵌在大纲行内，大纲可读性下降
3. **当前节点元数据缺失** — 没有显式说明当前节点 ID、路径、类型，AI 需要从 ★ 标记反推
4. **行为树语义隐含** — Seq/Sel 的执行规则只体现在数据裁剪中，AI 无法理解裁剪逻辑
5. **指令语句偏口语** — "你在解决这个问题之前" 这类表述增加解析负担

## 改动要点总结

| # | 改动 | 原因 |
|---|------|------|
| 1 | 加 Markdown 标题分隔各区块 | AI 可快速定位 `## 当前任务` / `## 已有知识` 等锚点 |
| 2 | 新增「当前任务」元数据表 | 显式提供节点ID、类型、执行路径，无需从树中反推 |
| 3 | 文档引用抽离到独立表格 | 知识大纲更干净；文档查找集中在一处 |
| 4 | 加行为树裁剪规则说明 | AI 理解为什么某些分支被省略 |
| 5 | 顶层标题 `# 行为树留言上下文` | 明确文档用途 |
| 6 | 指令语句精简为一行约束 | 减少解析负担 |
