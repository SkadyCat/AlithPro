# 交付说明 — 修复提案格式：节点数据/设定缺失

## 源任务文件

## 任务目标
迭代修复上一轮提出的 AI 友好留言模板格式 — 用户指出行为树节点上预制的 `data` 设定（如 AccountID=33123, Password=312312314123）在提案中丢失了。

## 问题分析
- 行为树节点可以携带 `data` 字段和 `status` 字段（如 `{AccountID:33123}`, `status:"done"`）
- 当前生产模板 (`app.js`) 也不展示这些数据，仅展示标题和文档引用
- 我的提案同样遗漏了这些数据
- 这些是用户通过 BT 执行顺序预制的关键设定，AI 必须知道

## 修复方案

### 知识大纲中内联节点 data

**原提案格式（缺少设定）：**
```
1.1 定义一些基础设定
  1.1.1 定义AccountID
  1.1.2 定义 NetID
  1.1.3 Password
```

**修复后格式（设定可见）：**
```
1.1 定义一些基础设定
  1.1.1 定义AccountID [✓] {AccountID: 33123}
  1.1.2 定义 NetID
  1.1.3 Password [✓] {Password: 312312314123}
  1.1.4 定义 友好值系统-Comp
  1.1.5 留言测试1 [✓]
```

### 规则说明
- `[✓]` = `status: "done"`，无 status 则不标注
- `{key: value}` = 节点的 `data` 字段内容，原样输出
- 无 data 的节点不展示花括号
