## Objective
Refactor the mind map message (留言) feature so that every message document includes the **execution chain** of the current node based on behavior tree rules (Seq/Sel/Action/Con).

## Changes Made

### `application/public/app.js`
1. **Added `dmBuildExecutionChain(targetNode)` function** (lines 519-553):
   - Walks from root to the target node, collecting the ancestor path.
   - For each ancestor on the path, renders all its children (siblings) so the agent sees prerequisite/successor steps.
   - Nodes on the active path are recursed into; sibling nodes are shown as leaves.
   - Marks `✅` for done nodes and `← 当前节点` for the target.

2. **Updated message content template** (lines 665-670):
   - Added `# 执行链路 (行为树)\n${dmBuildExecutionChain(node)}` section between the selected-node info and the user's message.
   - The full tree markdown (`# 思维导图结构`) is preserved for overview context.
