# 交付说明 — 留言内容迭代

## 源任务文件
`msg-2026-04-10T04-25-54-957Z.md`

## 任务目标
将留言文档内容格式从旧的三段式（思维导图结构 + 当前选中节点 + 执行链路）替换为新的"编号知识大纲"格式。新格式以用户消息为主导，展示当前已有知识作为先验上下文。

## 所做改动

### `application/public/app.js`
1. **删除 `dmBuildExecutionChain()`**（旧的扁平行为树执行链路函数）
2. **新增 `dmBuildKnowledgeOutline(targetNode)`**：
   - 跳过根节点（测试）
   - 根节点的首个子节点作为第一个 section header（`1. 友好值系统`）
   - section header 的第一个子节点被吸收为 sub-item（`1.1`），后续子节点各自成为新 section header（`2.`, `3.`, ...）
   - Sel 节点若在到达目标节点的路径上，则整体排除
   - 具有 messages 的节点附带 `需求文档: [file] 交付文档: [file_doc]` 标注
3. **更新 `onLeaveMessage` 内容模板**：
   - 新格式：`我们目前需要解决的是: {message} → 你已经具备以下知识 → 知识大纲 → 收束思考范围`
   - 移除了 `treeMarkdown`、`path` 变量的引用及旧的 `# 思维导图结构`、`# 当前选中节点`、`# 执行链路` 章节

### `drafts/draft-2026-04-10T03-21-14-013Z.md`
